## 一、研究问题

论文围绕三个核心研究问题展开：

| RQ | 问题 | 核心发现 |
|----|------|----------|
| **RQ1** | LLM叙事研究常用哪些叙事理论和数据集？ | 古典叙事学（形式主义、结构主义）最常用；数据集涵盖小说、短篇、新闻等 |
| **RQ2** | LLM在叙事任务中如何被使用，性能如何？ | 生成任务只用Prompt Engineering，理解任务用Fine-tuning；缺乏统一评估标准 |
| **RQ3** | NLP研究如何贡献于叙事学理论？ | 可规模化处理文本、综合理论、通过实验验证理论 |

## 二、核心理论框架

### 2.1 叙事学三层次模型

论文采用 Genette (1980) 的经典框架：

```
┌─────────────────────────────────────────────────────────────┐
│                      叙事层次模型                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Fabula (素材层)                                            │
│   ├─ 定义：按时间顺序排列的事件序列                           │
│   ├─ 关注：发生了什么                                        │
│   └─ 示例：提取社交网络、事件分割、角色分类                   │
│                                                             │
│   Discourse (话语层)                                         │
│   ├─ 定义：事件的呈现方式                                    │
│   ├─ 关注：如何讲述                                          │
│   └─ 示例：视角、时序、风格特征标注                          │
│                                                             │
│   Narration (叙述层)                                         │
│   ├─ 定义：叙述行为本身                                      │
│   ├─ 关注：谁在讲、对谁讲                                    │
│   └─ 示例：叙述者可靠性判断、道德提取                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 叙事学演进路线

```
古典叙事学 (Classical Narratology)
├─ 起源：俄国形式主义 + 法国结构主义
├─ 代表：Propp《民间故事形态学》、Genette《叙事话语》
├─ 关注：叙事结构、功能、语法
└─ 核心：fabula/sujet, histoire/récit, story/discourse
        ↓
后古典叙事学 (Post-classical Narratology)
├─ 认知叙事学 (Cognitive Narratology)
│   └─ 关注：心理处理、视角代入、共情
├─ 语境叙事学 (Contextual Narratology)
│   └─ 关注：文化、历史、意识形态
└─ 跨媒介叙事学 (Transmedial Narratology)
    └─ 关注：不同媒介的叙事策略
```

### 2.3 关键叙事学概念

| 概念 | 定义 | 来源 | NLP应用 |
|------|------|------|---------|
| **功能 (Function)** | 最小叙事单位 | Propp | 序列拆解、情节标注 |
| **视角 (Focalization)** | 信息呈现的角度 | Genette | 视角分类、信息控制 |
| **叙述者可靠性** | 叙述者是否可信 | Booth | 不可靠叙述者检测 |
| **叙事性 (Narrativity)** | 文本呈现叙事特征的程度 | Herman | 叙事/非叙事分类 |
| **行动元 (Actant)** | 角色的功能角色 | Greimas | 角色关系提取 |

## 三、NLP流水线与叙事理论的整合

### 3.1 三阶段整合

```
┌─────────────────────────────────────────────────────────────┐
│                    NLP流水线                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Stage 1: Labeling (标注)                                   │
│  ├─ 任务：人工标注数据集                                     │
│  ├─ 理论应用：                                              │
│  │   ├─ Genette视角理论 → 视角特征标注                      │
│  │   ├─ Keen共情理论 → 共情维度标注                         │
│  │   └─ Labov社会语言学 → 从句分类                          │
│  └─ 趋势：LLM辅助大规模标注                                  │
│                                                             │
│  Stage 2: Modeling (建模)                                   │
│  ├─ 任务：训练/使用LLM完成任务                               │
│  ├─ 理论应用：                                              │
│  │   ├─ Propp故事功能 → 叙事规划                            │
│  │   ├─ 三幕结构 → 故事生成                                 │
│  │   └─ fabula-discourse → 多Agent生成                      │
│  └─ 技术：Prompt Engineering + Fine-tuning                  │
│                                                             │
│  Stage 3: Evaluation (评估)                                 │
│  ├─ 任务：质量评估                                          │
│  ├─ 方法：                                                  │
│  │   ├─ 自动指标：BLEU, F1, BERTScore                       │
│  │   ├─ 人工评估：Torrance创意写作测试                       │
│  │   └─ LLM-as-Judge                                       │
│  └─ 挑战：LLM评估与人类评估相关性低                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 LLM技术分布

| 技术 | 应用场景 | 代表工作 | 趋势 |
|------|----------|----------|------|
| **Prompt Engineering** | 生成任务 | 几乎所有生成工作 | 主流方法 |
| **Fine-tuning** | 理解任务 | 特征分类、标注 | 辅助方法 |
| **Multi-Agent** | 复杂生成 | Yu et al. (2025) | 新兴方向 |

**关键发现**：
- 故事生成任务**几乎只用Prompt Engineering**，几乎没有监督学习或强化学习
- Fine-tuning主要用于**理解任务**（分类、标注）
- 多Agent系统是**新兴方向**，将fabula和discourse分离处理

## 四、叙事理论与任务映射

### 4.1 理论-任务对照表

| 理论类型 | 适用任务 | 代表研究 |
|----------|----------|----------|
| **古典叙事学** | 分割、嵌入、生成、角色标注、道德识别 | Hatzel & Biemann (2024), Yu et al. (2025) |
| **认知叙事学** | 交互标注、分类、基准测试、共情评估 | Shen et al. (2024), Harel-Canada et al. (2024) |
| **语境叙事学** | 分类、道德提取、新闻叙事提取 | Zhou et al. (2024), Hobson et al. (2024) |

### 4.2 叙事层次与任务

| 层次 | Fabula层任务 | Discourse层任务 | Narration层任务 |
|------|--------------|-----------------|-----------------|
| **理解** | 事件分割、角色分类、交互分类 | 特征标注、风格分析 | 道德提取、可靠性判断 |
| **生成** | 事件构建、角色规划 | 视角控制、时序设计 | 叙述者设置 |

## 五、数据集全景

### 5.1 数据类型分布

```
虚构叙事 (Fictional Narratives)
├─ 小说 (Novels)
│   ├─ 优势：长文本、角色历史丰富
│   └─ 用途：角色分析、主题研究、distant reading
├─ 短篇故事 (Short Stories)
│   ├─ 优势：长度适中、多样性
│   └─ 用途：生成评估、特征分析
├─ 民间故事 (Folktales)
│   ├─ 优势：跨文化比较
│   └─ 用途：道德价值研究、偏见分析
└─ 电影剧本 (Scripts)
    ├─ 优势：对话丰富、多模态潜力
    └─ 风险：训练数据污染

非虚构叙事 (Non-fictional Narratives)
├─ 新闻 (News)
│   └─ 用途：文化分析、框架识别
├─ Reddit帖子
│   └─ 用途：个人叙事、共情研究
└─ 维基百科
    └─ 用途：情节摘要、传记叙事
```

### 5.2 数据挑战

| 挑战 | 说明 | 现状 |
|------|------|------|
| 标注数据稀缺 | 需要专家标注 | 少量高质量数据 |
| 多语言缺失 | 多为英语 | 呼吁多语言语料 |
| 长文本标注 | 成本高 | 很少有标注的长文本 |
| 评估标准不一 | 无统一基准 | 难以比较模型 |

## 六、LLM叙事能力评估

### 6.1 生成能力

| 维度 | 发现 | 来源 |
|------|------|------|
| 创意性 | LLM声称有创意，但专家评估低于New Yorker | Chakrabarty et al. (2024) |
| 可控性 | 难以控制风格、角色、情节约束 | Wang et al. (2023) |
| 原创性 | 常产生套路化、陈词滥调情节 | 多项研究 |
| 评估悖论 | LLM评委偏好LLM生成内容 | 自我偏好偏差 |

### 6.2 理解能力

| 任务 | 性能 | 说明 |
|------|------|------|
| 视角分类 | 中等 | 可识别叙事视角 |
| 叙事性判断 | 与人类有差距 | 对抽象动作判断不同 |
| 事件因果 | 困难 | 因果推理能力不足 |
| 角色关系 | 可接受 | 需要结构化输入 |

### 6.3 模型选择趋势

**Workshop on Narrative Understanding 统计**：

| 年份 | Decoder-only (GPT等) | Encoder (BERT等) | 其他 |
|------|---------------------|------------------|------|
| 2021 | 少数 | 多数 | 有 |
| 2024 | 4/9 | - | - |
| 2025 | **10/10** | - | - |

**趋势**：Decoder-only LLM已成为叙事研究的绝对主流。

## 七、与我们的方案的对照

### 7.1 架构对比

| 维度 | 综述框架 | 我们的方案 | 吻合度 |
|------|----------|------------|--------|
| **层次模型** | Fabula → Discourse → Narration | L1 → L2 → L3 → L4 | ⭐⭐⭐⭐⭐ |
| **L1种子层** | (未明确) | 故事愿景文档 | ⭐⭐⭐ |
| **L2架构层** | Fabula层处理 | 精修大纲（序列/功能） | ⭐⭐⭐⭐⭐ |
| **L3叙事层** | Discourse层处理 | 章纲（视角/节奏/话语） | ⭐⭐⭐⭐⭐ |
| **L4渲染层** | 文本生成 | 正文生成 | ⭐⭐⭐⭐ |

**核心发现**：我们的四层模型与叙事学三层次模型高度契合！

### 7.2 映射关系

```
我们的模型          叙事学框架

L1 种子层     →    (创意输入，理论未覆盖)
                    ↓
L2 架构层     →    Fabula层
                    ├─ 序列/功能 = 事件结构
                    ├─ 人物行动元 = 角色功能
                    └─ 世界观 = 设定
                    ↓
L3 叙事层     →    Discourse层
                    ├─ 视角选择 = Focalization
                    ├─ 节奏控制 = 时序/时限
                    └─ 话语模式 = 话语类型
                    ↓
L4 渲染层     →    Narration层 + 文本实现
                    └─ 风格实现
```

### 7.3 差异分析

| 差异点 | 综述观点 | 我们的独特贡献 |
|--------|----------|----------------|
| 领域 | 通用叙事 | **网文专属** |
| 理论 | 叙事学 | **叙事学 + 网文商业理论** |
| L2专家 | 未提及多专家协作 | **三专家会议机制** |
| 映射库 | 未提出 | **效果→技术映射库** |
| 驱动模式 | 未区分 | **三种驱动流** |

## 八、关键启示

### 8.1 理论层面

1. **Fabula-Discourse分离是核心**
   - 我们L2处理Fabula（发生了什么）
   - 我们L3处理Discourse（如何讲述）
   - 这正是论文强调的关键分离

2. **叙事学概念可操作化**
   - 视角、节奏、话语模式都有明确的实现方式
   - 可以直接用于L3映射库设计

3. **多Agent是正确方向**
   - 论文提到Yu et al. (2025)的成功案例
   - 我们的L2专家会议正是多Agent架构

### 8.2 实践层面

1. **Prompt Engineering足够吗？**
   - 论文指出：生成任务几乎只用Prompt
   - 但我们可能需要Fine-tuning来注入网文专属知识

2. **评估仍是最大挑战**
   - 无统一基准
   - 需要设计网文专属评估指标

3. **LLM-as-Judge要谨慎**
   - 存在自我偏好偏差
   - 应结合人类评估

### 8.3 未来方向

论文建议的四个方向：

| 方向 | 说明 | 对我们的启示 |
|------|------|--------------|
| 超越叙事学 | 关注更广泛的叙事研究 | 可借鉴传播学、心理学 |
| 验证特定理论 | 实验验证叙事学假设 | 可设计爽点公式验证实验 |
| 开发特定指标 | 不追求统一基准，专注单维度 | 设计爽点密度、节奏评分等 |
| 研究预训练影响 | 叙事文本对LLM的影响 | 可研究网文预训练的效果 |

## 十一、总结

### 核心观点

1. **我们的四层模型与叙事学三层次高度契合**，理论框架是正确的

2. **L3的"效果→技术"映射**正是论文强调的"Discourse层处理"

3. **多专家Agent架构**符合论文指出的"多Agent生成"趋势

4. **评估是最大挑战**，需要设计专属指标

### 行动建议

1. **短期内**：将综述中的叙事学概念直接用于L3映射库设计

2. **中期**：设计网文专属评估指标体系

3. **长期**：通过实验验证我们的理论创新（三专家、三驱动、双库映射）
