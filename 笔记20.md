# 笔记20：世界书（World Book）深化设计

## 一、当前状态

世界书是关键词触发的知识注入系统。每位专家发言时，系统根据对话中的关键词匹配世界书条目，将匹配到的知识注入 prompt。

### 1.1 数据模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识 |
| `keys` | string[] | 主触发关键词（任一命中即触发） |
| `content` | string | 注入的知识文本 |
| `secondaryKeys` | string[] | 次要关键词（全部命中才触发） |
| `constant` | boolean | 是否恒定注入（忽略关键词） |
| `priority` | number | 排序权重，默认 10 |
| `position` | string | 注入位置（如 "before_char"） |
| `metadata` | Record | 扩展元数据 |

### 1.2 API 端点

```
GET    /api/projects/:projectId/worldbook              — 列表
POST   /api/projects/:projectId/worldbook              — 创建
GET    /api/projects/:projectId/worldbook/:entryId     — 详情
PUT    /api/projects/:projectId/worldbook/:entryId     — 更新
DELETE /api/projects/:projectId/worldbook/:entryId     — 删除
POST   /api/projects/:projectId/worldbook/commit       — 版本快照
GET    /api/projects/:projectId/worldbook/history      — 版本历史
POST   /api/projects/:projectId/worldbook/process-sequence — AI 自动更新
POST   /api/projects/:projectId/worldbook/auto-commit  — 自动提交
```

### 1.3 已实现

- 完整 CRUD + 版本管理（git 风格 commit）
- 关键词匹配检索（主键/次键/constant）
- 优先级排序
- 前端 WorldBook.vue：条目列表、创建/编辑表单、统计侧栏、版本管理
- Python 端：`WorldBookManager.processSequence()` 已接 LLM
- 9 个单测全部通过

### 1.4 待实现

- TypeScript `WorldBookManager.processSequence()` 是空桩（返回空结果）
- `STStyleWorldBook.revert()` 抛出 "not implemented"
- 注入位置 `position` 字段尚无消费逻辑
- 与 L3/L4 标签系统的联动（笔记19 §4.4 的 `tagBindings` 概念源于世界书）
- 世界书条目与标签库的双向引用

### 2.4 ST 的选择逻辑（`selectiveLogic`）

```
AND_ANY (0): Primary OR secondary 任一命中
NOT_ALL (1): Primary AND NOT ALL secondary
NOT_ANY (2): Primary AND NOT ANY secondary
AND_ALL (3): Primary AND ALL secondary
```

### 2.5 ST 的注入位置（`position`）

```
before(0) / after(1) / ANTop(2) / ANBottom(3) / atDepth(4) / EMTop(5) / EMBottom(6) / outlet(7)
```

### 2.6 ST 的书激活机制（多书并存）

ST 支持四种书来源同时激活：

1. **Global lore** — 用户在设置中多选全局激活的书
2. **Character lore** — 每个角色可绑定一本主书 + 多本辅助书
3. **Chat lore** — 每个聊天可绑定一本书
4. **Persona lore** — persona 绑定一本书

激活流程（`getSortedEntries()`）：
- 并行从四个来源加载条目
- 按来源优先级去重（高优先级来源的条目遮蔽低优先级）
- 按插入策略排序：evenly / character_first / global_first

扫描流程（`checkWorldInfo()`）：
- 从聊天消息构建 WorldInfoBuffer
- 计算 token 预算（`budget % * maxContext`，有 cap）
- 遍历排序后的条目，检查关键词匹配（支持正则）
- 支持递归扫描（激活的条目内容反馈回缓冲区继续匹配）
- 支持最小激活数（不够则加深扫描深度）
- 支持 timed effects（sticky / cooldown / delay 用消息计数追踪）

### 2.7 ST 的 UI 布局

整个 World Info 面板是一个侧边抽屉（`drawer-content`），包含：

- **全局激活区**：多选下拉（哪些书全局激活）+ 全局设置滑块
- **编辑器区**：选择编辑哪本书 + 搜索/排序 + 分页条目列表
- **条目编辑区**：完整字段表单，token 计数器，拖拽排序

每条条目的行内显示：拖拽手柄、展开/折叠、启用/禁用开关、状态标签（Constant 蓝色 / Normal 绿色）、位置下拉、Depth/Order/Probability 输入、移动/复制/删除按钮。

## 三、对照与取舍

### 3.1 可以直接抄的

| ST 概念 | 适配到我们的项目 |
|---------|-----------------|
| 多文件存储（`worlds/<name>.json`） | `data/projects/<id>/worldbooks/<bookId>.json` |
| 条目 schema 大部分字段 | 直接扩展 `WorldBookEntry` 接口 |
| `selectiveLogic` 四种模式 | 替代当前的简单 AND 逻辑 |
| `position` 8 种注入位置 | 替代当前的 string 类型 |
| sticky/cooldown/delay 机制 | 新增 timed effects 支持 |
| inclusion group 互斥激活 | 新增 group 互斥逻辑 |
| 递归扫描 | 新增递归激活机制 |
| token budget 控制 | 新增预算计算 |

### 3.2 需要适配的

| ST | 我们 |
|----|------|
| jQuery + Bootstrap 面板 | Vue 3 SFC 组件 |
| 角色/聊天/Persona 绑定 | 画布节点绑定（专家选择世界书） |
| 全局多选激活 | 画布上连线 or 专家节点下拉选择 |
| 侧边抽屉编辑器 | 独立页面（新窗口） |
| `data/worlds/` 全局目录 | `data/projects/<id>/worldbooks/` 项目内 |

### 3.3 不需要抄的

- ST 的 characterFilter（角色名/标签过滤）— 我们的专家节点就是角色
- ST 的 persona/chat lore 多来源 — 我们只有节点绑定一种来源
- ST 的 PNGLorebook 导入 — 暂时不需要
- ST 的 vectorized/embedding — 暂时不需要

**存储目录：** `data/projects/<projectId>/worldbooks/`

**扩展 WorldBookEntry（抄 ST）：**
