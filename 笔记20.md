# 笔记20：世界书（World Book）深化设计

## 一、最终实现状态（2026-05-10）

世界书是关键词触发的知识注入系统。参照 SillyTavern World Info 架构，实现了多书存储、关键词过滤注入、PNG 角色卡导入。

### 1.1 数据模型（WorldBookEntry 完整字段）

| 字段 | 类型 | 默认值 | 状态 | 说明 |
|------|------|--------|------|------|
| `id` | string | — | ✓ | 唯一标识 |
| `keys` | string[] | [] | ✓ | 主触发关键词 |
| `content` | string | "" | ✓ | 注入的条目文本 |
| `secondaryKeys` | string[] | [] | ✓ | 次要触发词 |
| `constant` | boolean | false | ✓ | 恒定注入 |
| `selective` | boolean | true | ✓ | 启用选择性逻辑 |
| `selectiveLogic` | enum | AND_ANY | ✓ | AND_ANY / NOT_ALL / NOT_ANY / AND_ALL |
| `priority` | number | 10 | ✓ | 排序权重 |
| `position` | string | before_char | ✓ | 注入位置（before_char/after_char/an_top/an_bottom/at_depth/em_top/em_bottom） |
| `disable` | boolean | false | ✓ | 禁用条目 |
| `comment` | string | "" | ✓ | 标题/备注 |
| `probability` | number | 100 | ✓ | 触发概率 % |
| `depth` | number | 4 | ✗ | 搜索深度（递归扫描未实现） |
| `group` | string | "" | ✓ | 互斥组名 |
| `groupWeight` | number | 100 | ✓ | 组内权重 |
| `sticky` | number | null | ✗ | 激活后保持 N 条消息 |
| `cooldown` | number | null | ✗ | 停用后冷却 N 条消息 |
| `delay` | number | null | ✗ | 延迟 N 条消息后激活 |
| `role` | string | "" | ✗ | 注入角色 |
| `caseSensitive` | boolean | false | ✓ | 大小写敏感 |
| `matchWholeWords` | boolean | false | ✓ | 全词匹配 |
| `excludeRecursion` | boolean | false | ✗ | 不可被递归激活 |
| `preventRecursion` | boolean | false | ✗ | 阻止递归扫描 |
| `delayUntilRecursion` | number | 0 | ✗ | 延迟到第 N 层递归 |
| `ignoreBudget` | boolean | false | ✗ | 无视 token 预算 |

### 1.2 存储结构

```
data/projects/<projectId>/
  worldbook.json              → 保留向后兼容（自动迁移到 worldbooks/default.json）
  worldbooks/                 → 多书目录
    default.json              → 默认书
    <bookId>.json             → 每本书独立文件
    <bookId>_commits.json     → 版本快照记录
```

每本书的 JSON 格式：
```json
{
  "name": "书名",
  "createdAt": "...",
  "updatedAt": "...",
  "entries": { "id": { ...entry... }, ... }
}
```

### 1.4 注入架构（关键实现）

```
Pipeline 启动
    │
    ├── routes/meeting.ts: 从 req.body 提取 worldbook_bindings
    ├── 从 worldbooks/ 加载所有书的条目 → worldbookEntries: Map<bookId, WorldBookEntry[]>
    ├── 构建 perNodeWorldBook: Map<nodeId, bookId>
    │
    └── Worker 执行时:
        ├── buildObjectContext(obj) → containerContext 文本
        ├── resolveWorldbook(nodeId, contextTokens, state):
        │   ├── 查 perNodeWorldBook → 找绑定的书
        │   ├── 按关键词过滤条目（主键/次键/selectiveLogic/constant/disable）
        │   ├── probability 随机判定
        │   ├── group 互斥选择（加权随机）
        │   ├── 按 position 分组排序
        │   └── 返回匹配条目的格式化文本
        └── ExpertContext.worldbook = 过滤后的文本
                │
                └── buildPrompt() → {worldbook} 替换
```

### 1.5 前端 WorldBook.vue 功能

- **多书选择器**：顶部下拉切换当前编辑的书，支持新建/删除
- **条目编辑**：完整字段表单（主键/次键/内容/选择性逻辑/位置/优先级/概率/互斥组/大小写/全词等）
- **高级设置**：sticky/cooldown/delay/递归/ignoreBudget（标注"尚未实现"，UI 设为 disabled）
- **导入**：支持 JSON 文件 + SillyTavern PNG 角色卡（解析 tEXt 块提取 character_book）
- **导出**：下载为 JSON 文件
- **保存到文档库**：将当前书存入文档库
- **拖放导入**：从文档库侧边栏拖世界书 JSON 到页面即可导入
- **自定义确认弹窗**：避免浏览器拦截原生 confirm()

### 1.8 踩过的坑

| 问题 | 根因 |
|------|------|
| 配置绑定不生效 | `routes/meeting.ts` 构造 `config` 时漏了 `worldbook_bindings` 字段 |
| 所有条目无差别注入 | 最初直接打平文本不调用 `getActiveEntries` |
| `outputSeq` 丢失 | 编辑 `sharedState` 时误闭合对象 |
| 蓝框不消失/拖放无效 | `dropEffect='copy'` 与侧边栏 `effectAllowed='move'` 冲突 |
| `confirm()` 被浏览器拦截 | 换成自定义弹窗 |
