# 系统设计文档

> 当前为原型阶段，边界情况未完全定义。后续迭代中逐步细化。

## 一、系统概述

AI 辅助网文创作系统。4 层瀑布模型：L1 种子层 → L2 架构层 → L3 叙事层 → L4 渲染层。B/S 架构，模块化插件体系。

```
L1 种子层 → L2 架构层 → L3 叙事层 → L4 渲染层
故事愿景     精修大纲      章纲/细纲      正文
```

三元数据支撑：
- **世界书**：当前状态（人物/伏笔/道具/势力）
- **RAG-历史回顾**：过往记忆（已生成内容的检索）
- **RAG-技法参考**：写作素材（映射规则/参考文本）

## 二、核心协议体系

### BaseRAGAgent
```python
class BaseRAGAgent(ABC):
    def retrieve(self, query: str, context: dict) -> list[RetrievedDoc]: ...
    def index(self, documents: list[Document]) -> None: ...
```
两个实例：`rag_history`（索引当前作品，动态）和 `rag_technique`（预构建静态库）。

### BaseWorldBook
```python
class BaseWorldBook(ABC):
    def get_active_entries(self, tokens: list[str]) -> list[Entry]: ...
    def update_entry(self, id: str, data: dict) -> None: ...
    def commit(self, message: str) -> str: ...
    def revert(self, hash: str) -> None: ...
```

### BaseLLMProvider
```python
class BaseLLMProvider(ABC):
    def invoke(self, prompt: str, **kw) -> str: ...
    def stream(self, prompt: str, **kw) -> Iterator[str]: ...
```

### BaseExpert
```python
class BaseExpert(ABC):
    def speak(self, outline: Outline, context: dict) -> ExpertOpinion: ...
```

### BaseMeetingProtocol
```python
class BaseMeetingProtocol(ABC):
    def get_speaking_order(self, context: dict) -> list[tuple[str, str]]: ...
    def should_continue(self, rounds: int, consensus: float) -> bool: ...
    def format_output(self, history: list[ExpertOpinion]) -> Outline: ...
```
三种内置：`character_driven`、`plot_driven`、`market_driven`。

### 各层策略
```python
class BaseSeedGenerator(ABC):
    def generate(self, user_input: dict) -> VisionDocument: ...

class BaseL2Architect(ABC):
    def generate_outline(self, vision, world_book, rag_history,
                         rag_technique, llm, human_feedback: Callable) -> Outline: ...

class BaseL3Narrative(ABC):
    def generate_chapter_plan(self, outline, rag_technique,
                              world_book) -> ChapterPlan: ...

class BaseL4Renderer(ABC):
    def render(self, plan, rag_technique, world_book) -> GeneratedText: ...
```

## 三、流水线编排

```
PipelineEngine.run(user_input, mode)
  ├─ L1.generate() → VisionDocument
  ├─ L2.generate_outline()
  │    ├─ MeetingProtocol.get_speaking_order()
  │    ├─ for each expert:
  │    │    Expert.speak() → SSE yield expert_speak
  │    │    if semi_auto: yield waiting_user → await human_feedback()
  │    ├─ auto-insert RAG/WorldBook → SSE yield rag_insert/worldbook_insert
  │    └─ format_output() → Outline
  ├─ L3.generate_chapter_plan() → ChapterPlan
  └─ L4.render() → SSE stream prose
```

## 五、API 设计

约 20 个端点。核心模式：REST POST/GET + SSE。

SSE 格式：`event: {type}\ndata: {json}\n\n`

事件类型：`expert_speak`、`user_message`、`rag_insert`、`worldbook_insert`、`waiting_user`、`error`、`done`

### 关键端点

```
项目管理
  POST   /api/projects
  GET    /api/projects/{id}
  PUT    /api/projects/{id}/config

L2 专家会议
  GET    /api/projects/{id}/l2/stream      SSE
  POST   /api/projects/{id}/l2/message     用户插话
  POST   /api/projects/{id}/l2/decide      用户决策
  POST   /api/projects/{id}/l2/continue    继续

L3 叙事层
  POST   /api/projects/{id}/l3/scenes      提交标签 → 生成细纲
  GET    /api/projects/{id}/l3/tags        获取标签库

L4 渲染层
  GET    /api/projects/{id}/l4/stream      SSE 生成正文

世界书
  GET    /api/projects/{id}/worldbook
  PUT    /api/projects/{id}/worldbook/{id}

RAG
  POST   /api/projects/{id}/rag/search

设置
  GET    /api/tags/l3, /api/tags/l4
```

## 七、L2 专家会议

### 群聊式界面

- 每个角色独立颜色和头像
- 用户随时插话
- RAG/世界书管理员自动插话
- 底部决策栏：通过/修改/驳回
- 右侧大纲实时预览

### 协作模式

| 模式 | 行为 |
|------|------|
| semi_auto | 专家发言后暂停，用户点"继续" |
| full_auto | 专家自动发言，用户插话切回半自动 |
| manual | 用户指定发言者 |

### 会议协议

| 协议 | 顺序 | 场景 |
|------|------|------|
| character_driven | 人物(main) → 架构(supplement) → 编辑(review) | 文青文 |
| plot_driven | 架构(main) → 编辑(review) → 人物(supplement) | 无限流 |
| market_driven | 编辑(main) → 架构(supplement) → 人物(review) | 商业定制 |

## 八、多维功能标注

L2 输出格式：
```
| # | 功能简述 | 剧情轴 | 人物轴 | 世界轴 | 读者效果 | 情绪 | 人物 | 说明 |
```

标注轴：
- 剧情轴（推进/转折/铺垫/阻碍/回收/承接）→ 架构师
- 人物轴（揭示特性/触发成长/确立关系/制造矛盾/展示能力/施加压力）→ 人物设计师
- 世界轴（展示设定/营造氛围/建立规则/拓展地图/强化画面感）→ 架构师+人物设计师
- 读者效果（期待感/满足感/代入感/爽感/惊喜感，标↑↓）→ 网络编辑

读者效果列直接成为 L3 映射输入：
```
爽感+爆发  → 外聚焦 + 短句动词密集
期待感+压抑 → 反派内聚焦 + 慢速扩述 + 大量对话
满足+余韵  → 自由间接引语 + 快速概述
```

## 十、世界书

四层结构：核心层 → 活跃层 → 归档层 → 索引层

条目格式（SillyTavern 风格 JSON）：
```json
{
  "id": "char_001", "keys": ["老周"], "secondary_keys": ["拍卖行"],
  "content": "老周，金丹期鉴定师...",
  "constant": false, "priority": 10, "position": "before_char"
}
```

关键机制：
- 版本链：不覆写，存 `[ch1-30:筑基]→[ch31-80:金丹]→[ch81-now:元婴]`
- 重要性衰减：权重 +1 被引用，-1 连续 10 章未引用，归零归档
- 自动压缩：已完成序列压缩为一行摘要

### 世界书管理员 Agent

每序列完成后后处理：读取→提取变化→更新→冲突检测→压缩归档→产出变更摘要。
实现：`backend/services/worldbook_manager.py`。

## 十一、RAG 系统

两个独立实例：
- RAG-历史回顾：索引当前作品，动态增长，多维度切片检索，Agent 压缩后注入
- RAG-技法参考：预构建静态库

实时性：序列完成 → 秒级关键词索引 + 异步向量索引（~25 分钟），检索自动选最佳。
实现：`backend/services/rag_indexer.py`。

## 十二、数据存储

| 数据 | 格式 | 说明 |
|------|------|------|
| 世界书 | JSON | 人类可读 |
| 配置 | JSON/YAML | 配置文件 |
| L1-L4 产出 | JSON/Markdown | 文档性质 |
| 日志/版本历史 | SQLite | 追加型、按时间查询 |

用户级（`data/user/`）跨项目共享，项目级（`data/projects/{name}/`）隔离。

## 十三、错误处理

| 模式 | 超时 | 失败 |
|------|------|------|
| semi_auto | 重试一次 | 聊天显示错误（不污染上下文），用户决定 |
| full_auto | 重试到底 | 自动重试 |

SSE 断线：EventSource 自动重连，前端保留已接收文本。用户选继续（传断点续写）或重来。

通信：REST POST/GET + SSE。浏览器原生 EventSource。
