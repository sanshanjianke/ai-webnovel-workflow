# 系统设计文档

## 一、系统概述

本系统将网文创作过程工程化，采用 **B/S 架构**，前后端分离。核心设计理念是**模块化、可配置、可扩展**——每个功能组件定义统一协议，具体实现注册到系统，用户通过配置文件或前端面板切换。

### 四层创作模型

```
L1 种子层 → L2 架构层 → L3 叙事层 → L4 渲染层
故事愿景     精修大纲      章纲/细纲      正文
```

### 三元数据支撑

```
世界书（当前状态）  +  RAG-历史回顾（过往记忆）  +  RAG-技法参考（写作素材）
```

---

## 二、技术选型

| 项目 | 选型 | 理由 |
|------|------|------|
| 后端语言 | Python | LangChain、向量数据库等生态 |
| 后端框架 | FastAPI | 原生 async/await，SSE 内置支持 |
| 前端框架 | Vue | 模板语法易上手，中文文档好 |
| 实时通信 | SSE | 单向推送匹配 LLM 流式输出 |
| 向量数据库 | Chroma | 嵌入式运行，无需独立服务 |
| 数据存储 | JSON + SQLite | JSON 存核心数据，SQLite 存日志 |
| 部署方式 | `start.sh`/`start.bat` | 双击启动，参考 SillyTavern 模式 |

### 前端技术细节

- Markdown 渲染：`markdown-it`（Vue 生态）
- 流式渲染：暂存未闭合的 Markdown 块，语法完整后再渲染
- 实时通信：浏览器原生 `EventSource`，自带重连
- 标签拖拽：采用拖拽库将标签从面板拖入指令文本区

## 三、模块化协议体系

### 3.1 协议总览

```
RAG 模块          ──  BaseRAGAgent
世界书模块         ──  BaseWorldBook
LLM 调用          ──  BaseLLMProvider
L2 专家           ──  BaseExpert
L2 会议结构        ──  BaseMeetingProtocol
L1 种子生成       ──  BaseSeedGenerator
L2 大纲生成       ──  BaseL2Architect
L3 叙事            ──  BaseL3Narrative
L4 渲染            ──  BaseL4Renderer
```

### 3.2 核心协议定义

#### BaseRAGAgent

```python
class BaseRAGAgent(ABC):
    @abstractmethod
    def retrieve(self, query: str, context: dict) -> list[RetrievedDoc]:
        """检索：输入查询+当前上下文 → 返回文档列表"""
        ...

    @abstractmethod
    def index(self, documents: list[Document]) -> None:
        """索引：输入文档 → 建立索引"""
        ...
```

#### BaseWorldBook

```python
class BaseWorldBook(ABC):
    @abstractmethod
    def get_active_entries(self, context_tokens: list[str]) -> list[Entry]:
        """根据当前上下文触发词，返回应注入的条目"""
        ...

    @abstractmethod
    def update_entry(self, entry_id: str, data: dict) -> None:
        """更新单条条目"""
        ...

    @abstractmethod
    def commit(self, message: str) -> str:
        """保存当前状态为一个版本 → 返回 commit hash"""
        ...

    @abstractmethod
    def revert(self, commit_hash: str) -> None:
        """回滚到指定版本"""
        ...
```

#### BaseLLMProvider

```python
class BaseLLMProvider(ABC):
    @abstractmethod
    def invoke(self, prompt: str, **kwargs) -> str:
        """同步调用 LLM，返回完整响应"""
        ...

    @abstractmethod
    def stream(self, prompt: str, **kwargs) -> Iterator[str]:
        """流式调用 LLM，逐 token 返回"""
        ...
```

#### BaseExpert

```python
class BaseExpert(ABC):
    @abstractmethod
    def speak(self, outline: Outline, context: dict) -> ExpertOpinion:
        """根据当前大纲和上下文生成专家发言"""
        ...
```

#### BaseMeetingProtocol

```python
class BaseMeetingProtocol(ABC):
    @abstractmethod
    def get_speaking_order(self, context: dict) -> list[tuple[str, str]]:
        """返回发言顺序 [(expert_id, speech_type), ...]
        speech_type: "main" | "review" | "supplement"
        """
        ...

    @abstractmethod
    def should_continue(self, rounds: int, consensus: float) -> bool:
        """判断会议是否继续迭代"""
        ...

    @abstractmethod
    def format_output(self, history: list[ExpertOpinion]) -> Outline:
        """将会议记录整理为 L3 可接收的大纲格式"""
        ...
```

### 3.3 各层策略的多种实现

| 层 | 实现 | 说明 |
|----|------|------|
| **L1** | `GuidedFormGenerator` | 引导式表单 → 填充模板（当前方案） |
| | `ConversationalGenerator` | 对话式逐步提取 |
| | `OneShotGenerator` | 一次输入长文本，AI 提取结构 |
| **L2** | `ExpertMeetingL2` | 三专家会议（当前方案） |
| | `SinglePassL2` | 单次生成，快速出稿 |
| **L3** | `MappingCompilerL3` | 效果→叙事指令查表映射（当前方案） |
| | `DirectInferenceL3` | LLM 直接推断叙事指令 |
| **L4** | `ConstrainedRenderer` | 严格按 L3 约束生成（当前方案） |
| | `FreeRenderer` | L3 只给建议，L4 有更多自由度 |

## 四、L2 专家会议系统

### 4.1 三种驱动模式（会议结构）

| 模式 | 发言顺序 | 适用场景 | 最大轮次 |
|------|---------|---------|---------|
| `character_driven` | 人物设计师 → 剧情架构师 → 网络编辑 | 文青文、种田文 | 3 |
| `plot_driven` | 剧情架构师 → 网络编辑 → 人物设计师 | 无限流、系统文 | 3 |
| `market_driven` | 网络编辑 → 剧情架构师 → 人物设计师 | 跟风热点、商业定制 | 4 |

会议协议采用插件目录自动发现机制。新增协议只需将 `.py` 文件放入 `meeting_protocols/` 目录，系统扫描并注册。配置文件只声明选择哪个，不枚举有哪些可选。

### 4.2 群聊式交互界面

L2 采用**群聊式界面**（类似 Discord/Slack），多角色协作会议模型：

```
┌─────────────────────────────────────────────────────────
│ L2 专家会议                            [半自动 ▾] [暂停]
├─────────────────────────────────────────────────────────
│  各角色（颜色+头像区分）：专家发言、用户插话、
│  RAG Agent 和世界书管理员自动插话
│  ──────────────────────────────────────────
│  [通过 ✓] [要求修改 ✎] [驳回 ↩]
├─────────────────────────────────────────┤
│ 右侧面板：实时大纲预览                     │
└─────────────────────────────────────────┘
```

交互规则：
- 每个角色独特头像和颜色标识
- 用户随时插话，输入框始终可用
- 专家可 `@` 点名其他专家
- RAG Agent 和世界书管理员自动插入信息（不等用户触发）
- 底部决策按钮控制流程走向

### 4.3 三种协作模式

| 模式 | 行为 |
|------|------|
| **半自动** | 每位专家发言后暂停，用户点击"继续"进入下一位 |
| **全自动** | 专家自动发言，用户纯旁观。用户打字插话自动切回半自动 |
| **手动** | 用户指定哪位专家发言 |

## 五、L3/L4 标签化指令系统

### 5.1 设计理念

L1/L2 以文本对话为主。L3/L4 的产物是结构化写作指令，适合用标签编排。用户有两种修改方式：
1. 直接编辑指令文本
2. 从标签库拖拽标签到文本中

### 5.2 L3 标签库（叙事指令）

- **视角标签**：内聚焦（反派/主角/路人）、外聚焦、零聚焦、自由间接引语、不定内聚焦
- **节奏标签**：慢速扩述、中速等述、快速概述、停顿、省略
- **信息控制标签**：减少信息·制造悬念、增加信息·内心独白、限制视角、叙事空白
- **话语模式标签**：大量对话、动作描写为主、心理描写为主、环境烘托为主、对话+动作交替
- **情绪锚点标签**：压抑积累、爆发释放、余韵回味、紧张渐进、温馨舒缓
- **智能推荐区**：AI 根据当前场景自动推荐标签

### 5.3 L4 标签库（渲染技法）

- **风格标签**：古龙式冷硬对话、海明威式动作描写、修真文言风格、轻松网路口语、严肃史诗叙事
- **句式标签**：短句动词密集、长短句交替、排比增强气势、省略主语制造紧张
- **描写技法标签**：五感描写、环境参与叙事、只写动作不写心理、侧面烘托、陌生化
- **叙事转文标签** ★：将叙事术语直接转换为具体文本模板
  - "震惊反应" → 人物失态+环境凝滞模板
  - "压抑氛围" → 环境阴暗+心理沉重模板
  - "装逼时刻" → 外聚焦+短句+冷淡反应模板
  - "战斗爆发" → 动词链+快节奏切换模板
- **参考范例标签**：拖入即插入范文片段

### 5.4 标签数据结构

```json
{
  "id": "focal_internal_villain",
  "name": "内聚焦·反派视角",
  "category": "视角",
  "layer": "L3",
  "prompt": "使用内聚焦视角，以反派角色的认知范围呈现场景...",
  "conflicts": ["零聚焦·全知叙述", "外聚焦·客观镜头"],
  "synergies": ["慢速扩述·细节展开", "大量对话·对峙冲突"],
  "example": "他看着那个穷小子，嘴角勾起一抹讥笑..."
}
```

标签间关系：
- **conflicts**：互斥标签
- **synergies**：推荐搭配
- **prompt**：实际注入的指令文本，高级用户可展开修改

## 六、功能的多维标注体系

### 6.1 标注矩阵

每个叙事功能从对内和对外两个大维度标注：

```
对内（故事世界内部）
├── 剧情轴：推进 / 转折 / 铺垫 / 阻碍 / 回收 / 承接
├── 人物轴：揭示特性 / 触发成长 / 确立关系 / 制造矛盾 / 展示能力 / 施加压力
└── 世界轴：展示设定 / 营造氛围 / 建立规则 / 拓展地图 / 强化画面感

对外（读者心理 / 网文五感）
├── 期待感：悬念期待 / 目标期待 / 反转期待
├── 满足感：伏笔回收 / 压抑释放 / 公平回报
├── 代入感：感官代入 / 情感代入 / 认知代入
├── 爽感：装逼 / 打脸 / 获得 / 碾压 / 崇拜
└── 惊喜感：反转 / 揭示 / 新意
```

### 6.2 与 L2 三专家的职责对齐

- 剧情轴 → 剧情架构师
- 人物轴 → 人物设计师
- 世界轴 → 剧情架构师 + 人物设计师
- 读者五感 → 网络编辑

### 6.3 L2 输出格式

```
| # | 功能简述 | 剧情轴 | 人物轴 | 世界轴 | 读者效果 | 情绪 | 人物 | 说明 |
|---|---|---------|--------|--------|--------|---------|------|------|------|
```

新增列均为多选，读者效果可标注强度（↑↓）。

### 6.4 对 L3 的收益

"读者效果"列直接成为 L3 的映射输入，省去 L3 推断读者效果这一步：

```
爽感+爆发  → 外聚焦视角 + 短句动词密集
期待感+压抑 → 反派内聚焦 + 慢速扩述 + 大量对话
满足+余韵  → 自由间接引语 + 快速概述
```

### 6.5 与 Propp 的衔接

Propp 的六种功能作为剧情轴的快捷标签保留：
- 获得 → 推进 | 禁令 → 铺垫 | 违背 → 转折 | 斗争 → 阻碍/推进 | 胜利 → 转折/回收 | 失败 → 阻碍

## 七、世界书系统

### 7.1 四层结构

```
世界书
├── 核心层（永久）    世界观规则、核心设定、阅读契约
├── 活跃层（实时更新） 人物状态、活跃伏笔、道具、势力状态
├── 归档层（摘要压缩） 已完成序列摘要、已回收伏笔、退场人物、版本历史
└── 索引层（元数据）   条目-章节映射、条目间关系图、触发词索引
```

### 7.2 条目格式（参考 SillyTavern）

```json
{
  "id": "char_lizhou_001",
  "keys": ["老周", "周师傅"],
  "secondary_keys": ["拍卖行", "青云城"],
  "content": "老周，金丹期鉴定师...",
  "constant": false,
  "priority": 10,
  "position": "before_char",
  "selective": true
}
```

### 7.3 版本链

角色状态变化不覆写，保留完整版本历史：

```
主角修为: [第1-30章: 筑基期（描述）]
          [第31-80章: 金丹期（突破原因）]
          [第81章至今: 元婴期（突破原因）]
```

### 7.4 重要性衰减

- 初始权重取决于角色重要性
- 每次被创作引用 +1
- 连续10章未引用 -1
- 权重低于阈值：不出现在 L4 上下文，但保留在活跃层
- 权重归零 → 归档

### 7.5 自动压缩

已完成序列从详细描述压缩为摘要：

```
归档层存储: "序列03-拍卖会打脸[第12-14章]: 主角展示实力，
获300灵石，结识老周，埋京城伏笔。打脸对象：赵少。"
```

## 八、世界书管理员 Agent

### 定位

元层 Agent，不参与创作讨论。在每轮创作完成后做后处理。

### 职责

1. 读取当前序列/章节产出
2. 提取：新人物、状态变化、新地点、道具、伏笔、关系变化
3. 更新对应条目
4. 冲突检测：新内容是否与旧设定矛盾 → 有矛盾提醒用户
5. 压缩归档：旧条目是否需要合并/归档
6. 产出：更新后的世界书 + 变更摘要

### 触发词自维护

- 发现某条目触发词总在不该激活时激活 → 增加触发条件
- 发现某条目应该被激活却未激活 → 补充触发词
- 两个条目触发词高度重叠 → 考虑合并

## 九、RAG 系统

### 9.1 双重职能

| 职能 | RAG-历史回顾 | RAG-技法参考 |
|------|-------------|-------------|
| 内容 | 当前作品的已生成内容 | 爽点公式、映射规则、参考文本 |
| 数据来源 | 动态增长 | 预构建静态库 |
| 解决的问题 | "之前发生了什么" | "怎么写更好" |
| 更新频率 | 每完成序列即索引 | 很少更新 |

### 9.2 历史回顾的索引策略

利用笔记7的多维度切片（plot/character/emotion/function）：

```
角色"老周"再出场时：
  ├── character维度 → 检索所有出场片段
  ├── plot维度     → 检索参与过的序列摘要
  └── 世界书       → 当前状态
```

检索结果经 Agent 压缩后注入上下文。

### 9.3 实时性策略

```
序列完成
    ├── 立即：写入临时关键词倒排索引（秒级，低精度）
    └── 异步：启动后台向量索引 pipeline（约25分钟，高精度）
检索优先级：向量索引已覆盖 → 向量；未覆盖 → 关键词
```

## 十、LLM 调用层

统一内部接口 `BaseLLMProvider`，屏蔽各模型 API 差异。前端提供**模型配置页面**，用户自行填写 API 地址和密钥。

- **多模型适配**：OpenAI 兼容接口 / GLM / 本地模型
- **速率限制**：内置排队和退避
- **失败重试**：主模型失败自动切备用
- **Token 计数**：显示"本次花费 ¥0.38"
- **流式封装**：统一 `stream()` 事件流

## 十一、错误处理

策略由协作模式决定：

| 模式 | 超时重试 | 失败后 |
|------|---------|--------|
| **半自动** | 重试一次 | 群聊显示错误（不计入上下文），用户决定跳过/重试 |
| **全自动** | 重试直到成功 | 自动重试，无人值守 |

流式中断：
- 浏览器 EventSource 自动重连
- 前端保留已接收文本
- 用户选择：继续（传断点续写）或重来（删除重新生成）

## 十三、API 设计

约 20 个端点，分以下几组：

### SSE 事件流格式

```
event: {type}
data: {json}

```

事件类型包括：`expert_speak`、`user_message`、`rag_insert`、`worldbook_insert`、`waiting_user`、`error`

### 端点分组

```
项目管理
  POST   /api/projects                    创建项目
  GET    /api/projects/{id}               获取项目状态
  PUT    /api/projects/{id}/config        更新配置

L1 种子层
  POST   /api/projects/{id}/l1/generate   提交创意 → 故事愿景

L2 专家会议
  GET    /api/projects/{id}/l2/stream     SSE 流式推送专家发言
  POST   /api/projects/{id}/l2/message    用户发消息/插话
  POST   /api/projects/{id}/l2/decide     用户决策（通过/修改/驳回）
  POST   /api/projects/{id}/l2/continue   半自动模式点"继续"

L3 叙事层
  GET    /api/projects/{id}/l3/outline    获取当前大纲
  POST   /api/projects/{id}/l3/scenes     提交标签指令 → 生成细纲
  GET    /api/projects/{id}/l3/tags       获取标签库

L4 渲染层
  GET    /api/projects/{id}/l4/plan       获取当前细纲
```

## 十五、前端页面结构

共 7 个页面：

1. **项目仪表盘**：项目列表、创建新项目、进度概览
2. **L1 种子页**：引导式表单，AI 辅助展开
3. **L2 专家会议页**：群聊式专家会议 + 大纲预览（核心交互）
4. **L3 叙事页**：细纲编辑器 + 标签面板
5. **L4 渲染页**：正文生成 + 用户反馈
6. **世界书管理页**：树形浏览器 + 条目编辑器 + 版本历史
7. **设置页**：模块选择、API 配置、提示词模板

### 长篇小说导航

- 左侧章节树形目录，可折叠和跳转
- 点击某章加载该章的世界书快照
- 已完成章节可配置是否允许修改

### 导入导出

- 导出：正文 txt/markdown/epub，大纲导出，世界书导出
- 导入：已有小说 → 自动分章 → 生成世界书初稿 → 建 RAG 索引

### 初始引导

- 创建向导：选类型 → 选驱动模式 → 初始化世界书核心层
- 预置示例模板，降低空项目门槛
