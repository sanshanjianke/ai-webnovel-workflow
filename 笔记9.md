# 笔记9：功能的多维拆解模型

## 一、问题的起源

目前L2架构师所用的"功能"概念，沿用Propp的古典叙事学分类：

```
获得功能、禁令功能、违背功能、斗争功能、胜利功能、失败功能
```

这六种都是**纯动作标签**——只描述了"角色做了什么"，不关心"这个动作达成了什么效果"。

笔记3和笔记4中已经出现了直觉上的突破。笔记4对《纳萨力克降临》的功能拆解中，出现了"核心爽点功能"、"制造压抑感"、"侧面描写"等超出Propp框架的描述。但当时没有系统化。

现在重新审视：一个叙事单位（功能），应该从**对内**和**对外**两个大维度进行标注。对内管"故事内部发生了什么变化"，对外管"读者产生了什么反应"。

---

## 二、完整标注矩阵

对于一个功能，建议标注以下维度：

```
功能标注矩阵

对内（故事世界内部）
├── 剧情轴：这个功能对情节推进起什么作用？
│   ├── 推进：推动主线向前（做出决定、采取行动）
│   ├── 转折：改变情节方向（发现真相、意外事件）
│   ├── 铺垫：为后续事件埋因（引入伏笔、建立条件）
│   ├── 阻碍：制造障碍拖延满足（敌人出现、条件不足）
│   ├── 回收：兑现之前的铺垫（伏笔触发、条件成熟）
│   └── 承接：衔接前后事件（过渡、总结、反应）
│
├── 人物轴：这个功能对角色起什么作用？
│   ├── 揭示特性：展示角色的某个性格面（勇敢/怯懦/机智/愚蠢）
│   ├── 触发成长：角色的能力或认知发生改变（突破、领悟、黑化）
│   ├── 确立关系：角色间关系建立或改变（结盟、背叛、爱慕、仇恨）
│   ├── 制造矛盾：引发角色间冲突（误解、利益冲突、价值观对立）
│   ├── 展示能力：呈现角色的实力或资源（战斗、谋划、财力）
│   └── 施加压力：逼迫角色做选择（两难处境、倒计时、威胁）
│
└── 世界轴：这个功能对世界观呈现起什么作用？
    ├── 展示设定：展现世界规则或设定（力量体系、社会结构）
    ├── 营造氛围：建立场景的情绪基调（紧张、温馨、诡异、庄严）
    ├── 建立规则：明确或暗示世界运行逻辑（因果律、代价机制）
    ├── 拓展地图：引入新地点/势力/信息（解锁新区域）
    └── 强化画面感：通过感官描写增强沉浸（视觉/听觉/嗅觉/触觉）

对外（读者心理层面/网文五感）
├── 期待感：让读者想知道接下来会发生什么
│   ├── 悬念期待：留下未解之谜（谁杀了X？门后面是什么？）
│   ├── 目标期待：让读者想让主角达成某事（一定要赢！）
│   └── 反转期待：暗示表象不简单（这不对劲……）
│
├── 满足感：兑现读者的等待和预期
│   ├── 伏笔回收：之前的线索得到解释（原来那个玉佩是这个意思）
│   ├── 压抑释放：积累的负面情绪得到宣泄（终于打脸了！）
│   └── 公平回报：努力得到应有结果（付出终于有了收获）
│
├── 代入感：让读者进入角色的感知和情绪
│   ├── 感官代入：通过五感描写让读者"身临其境"
│   ├── 情感代入：让读者与角色共情（心疼、激动、愤怒）
│   └── 认知代入：让读者用角色的逻辑思考（如果是我该怎么办？）
│
├── 爽感：直接的情绪爆发和快感
│   ├── 装逼：主角展示远超他人预期的实力/财富/见识
│   ├── 打脸：轻视主角的人被事实碾压
│   ├── 获得：主角得到宝贵物品/能力/信息（捡宝/升级/觉醒）
│   ├── 碾压：主角轻松击败强大对手
│   └── 崇拜：他人对主角表现出的尊敬/畏惧/狂热
│
└── 惊喜感：打破读者的预期
    ├── 反转：情节方向出人意料（友方是卧底、弱者是大佬）
    ├── 揭示：隐藏信息的公开（真相大白、身份揭晓）
    └── 新意：跳出套路的意外处理方式（不按常规出牌）
```

## 三、标注示例

**场景**："拍卖会上，鉴定师老周接过主角的晶核，仪器瞬间爆表"

### 传统Propp标注

> 功能类型：获得功能（主角获得老周的认可和后续交易机会）

### 多维标注

```
功能：仪器爆表
├── 对内
│   ├── 剧情：转折（打破质疑僵局）+ 铺垫（引出后续交易）
│   ├── 人物：揭示特性（老周的专业→震惊）、确立关系（老周从质疑→认可主角）
│   └── 世界：展示设定（修真晶核的能量等级远超此界认知）
├── 对外
│   ├── 期待感：目标期待（读者想知道这晶核值多少钱）
│   ├── 满足感：压抑释放（之前被嘲讽的憋屈在此刻开始释放）
│   ├── 爽感：打脸的前奏（物理上的"爆表"是最直观的打脸证据）
│   └── 惊喜感：新意（不是老周口头服软，而是仪器爆表——客观事实胜过千言万语）
```

### 对比

传统标注只告诉我们这是一个"获得功能"。新标注告诉我们：

- 它在剧情上同时是**转折+铺垫**（Propp只取其一，实际上一个功能可以有多重剧情作用）
- 在人物上服务于老周这个角色的弧光（从怀疑到信服的变化过程）
- 在世界上展示了力量体系的差异（境界压制的设定展示）
- 在读者层面同时触发了四种感受（期待、满足、爽感、惊喜）
- 这个功能是整个打脸序列的**"证据锚点"**——不是靠嘴说，而是靠客观事实打脸，比任何台词都有说服力

## 四、序列的功能继承与聚合

单个功能被标注后，序列层面的功能可以由其组成功能聚合而成：

```
序列："拍卖会打脸"
组成功能：
├── F1 入场被嘲 → 剧情：铺垫 / 人物：制造矛盾 / 世界：展示设定 / 对外：期待感
├── F2 鉴定质疑 → 剧情：阻碍 / 人物：施加压力 / 世界：营造氛围 / 对外：期待感↑
├── F3 仪器爆表 → 剧情：转折 / 人物：确立关系 / 世界：展示设定 / 对外：爽感+满足感
├── F4 全场震惊 → 剧情：承接 / 人物：展示能力 / 世界：营造氛围 / 对外：爽感
└── F5 天价成交 → 剧情：推进+回收 / 人物：确立关系 / 世界：拓展地图 / 对外：满足感+期待感

序列级聚合：
├── 剧情定位：以转折为核心、前后有铺垫和回收的完整因果闭环
├── 人物作用：主角人设展示（深藏不露）+ 老周弧光（从质疑到追随）+ 赵少功能（敌对者被打脸退场）
├── 世界作用：力量体系展示（晶核=跨界产物）+ 拍卖行环境（末世中的秩序节点）
└── 读者效果：期待→压抑→爆发→满足的完整V型情绪曲线，爽感密度高，结尾留有新期待
```

## 五、与L2三专家的职责对齐

多维标注与三专家的专业领域天然对应：

```
功能标注矩阵          负责专家

对内/剧情轴    →    剧情架构师（逻辑推演、结构完整）
对内/人物轴    →    人物设计师（人设一致、行为合理）
对内/世界轴    →    剧情架构师（设定呈现） + 人物设计师（氛围感知）

对外/期待感    →    网络编辑（读者留存的核心指标）
对外/满足感    →    网络编辑（爽点公式的兑现，防虎头蛇尾）
对外/代入感    →    网络编辑（防出戏，防"翻译腔"）
对外/爽感      →    网络编辑（网文的核心商业价值）
对外/惊喜感    →    网络编辑（防套路化，维持新鲜感）
```

### 专家会议的新协作模式

1. **剧情架构师**拆解序列 → 对每个功能标注**剧情轴**和**世界轴**
2. **人物设计师**审阅 → 标注**人物轴**，审核人设一致性
3. **网络编辑**审阅 → 标注**读者五感**，评估情绪曲线是否达标
4. 同一功能如果有某轴空白 → 提示该功能的"潜台词"未被充分挖掘

### 多重标注的价值

一个功能在多个轴上同时产生作用，这是好剧情的特征——**一石多鸟**。

如果某个功能只在剧情轴上有"推进"，其他轴空白 → 这是一个"功能性"但"不好看"的功能，网络编辑应该提出改进建议。

如果某个读者效果在多个连续功能中都缺失 → 该序列存在情绪断层，可能需要重新调整。

## 六、L2输出格式升级

### 当前格式（L2.md）

```
| 序号 | 功能 | 情绪 | 人物 | 说明 |
|------|------|------|------|------|
| 1    | ...  | 压抑 | ...  | ...  |
```

问题：只有一行"功能"和"情绪"，无法表达一个功能的多维效果。

### 升级后的多维格式

```
| # | 功能简述 | 剧情轴 | 人物轴 | 世界轴 | 读者效果 | 情绪 | 人物 | 说明 |
|---|---------|--------|--------|--------|---------|------|------|------|
| 1 | 入场被嘲 | 铺垫   | 制造矛盾 | 展示设定 | 期待感   | 压抑 | 赵少 | 建立敌对关系 |
| 2 | 鉴定质疑 | 阻碍   | 施加压力 | 营造氛围 | 期待感↑  | 压抑↑| 老周 | 专业权威的质疑更具压迫感 |
| 3 | 仪器爆表 | 转折   | 确立关系 | 展示设定 | 爽感+满足| 爆发 | 老周 | 物理证据胜过千言万语 |
| 4 | 全场震惊 | 承接   | 展示能力 | 营造氛围 | 爽感     | 爆发 | 众人 | 侧面烘托主角逼格 |
| 5 | 天价成交 | 推进+回收| 确立关系| 拓展地图 | 满足+期待| 余韵 | 主角 | 物质收获+埋下一章钩子 |
```

新增列说明：

| 新增列 | 可选值（可多选） | 强度标注 |
|--------|-----------------|---------|
| 剧情轴 | 推进/转折/铺垫/阻碍/回收/承接 | — |
| 人物轴 | 揭示特性/触发成长/确立关系/制造矛盾/展示能力/施加压力 | — |
| 世界轴 | 展示设定/营造氛围/建立规则/拓展地图/强化画面感 | — |
| 读者效果 | 期待感/满足感/代入感/爽感/惊喜感 | 可标 ↑↓ |

## 七、与L3的衔接收益

多维标注直接降低了L3的工作难度：

```
L3的核心任务：效果指令 → 叙事技术指令

原来L3的工作模式：
1. 读取功能简述 + 情绪标签
2. 自己推断这个功能产生了什么读者效果 → 不可靠，依赖AI推理能力
3. 根据推断的效果查找映射规则
4. 生成叙事指令

多维标注后：
1. 直接读取"读者效果"列 → 目标明确
2. 查映射表即可：
     爽感+爆发  → 外聚焦视角 + 短句动词密集
     期待感+压抑 → 反派内聚焦 + 慢速扩述 + 大量对话
     满足+余韵  → 自由间接引语 + 快速概述
3. 生成叙事指令
```

省去的是L3最不可靠的一步——**推断读者效果的逆向工程**。

## 八、与Propp功能的衔接

Propp的六种经典功能不是被废弃，而是作为**剧情轴的快捷标签**继续存在：

```
Propp功能      →  对应剧情轴标签

获得功能       →  推进（通常）
禁令功能       →  铺垫（通常，为违背做铺垫）
违背功能       →  转折（通常）
斗争功能       →  阻碍或推进（取决于结果）
胜利功能       →  转折或回收（通常）
失败功能       →  阻碍（通常，为后续胜利做铺垫）
```

Propp功能可以作为**快捷标注**，AI或用户可以先选Propp类型，系统自动推导一个默认的剧情轴标签，然后再手动或自动补充人物轴、世界轴和读者效果。

## 九、待讨论问题

1. **自动化标注的可行性**：多维标注是手动标注还是可以由AI自动完成？如果AI自动标注，准确率能到多少？
2. **标注粒度**：是否所有功能都需要完整标注所有轴？还是只标注"主要效果"？
3. **读者效果的量化**：期待感↑和期待感↑↑之间如何定义？是否需要更细的量化标准（如1-5级）？
4. **与情绪曲线的对应**：读者效果与情绪（压抑/爆发/余韵）之间是否存在固定的对应关系？
5. **跨序列的效果追踪**：如何确保一个序列的读者效果与前后序列衔接？是否需要序列间的效果过渡规则？
6. **标注数据积累**：如果将多维标注作为训练数据，需要多少标注样本才能训练一个可靠的功能分类模型？
