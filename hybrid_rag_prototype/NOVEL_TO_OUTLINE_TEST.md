# 笔记6混合检索方案 - 小说转大纲测试总结

## 核心发现

### 问题识别

原方案基于L2层产物（大纲/细纲），但测试时使用的是完整小说文本。这导致：
1. 无法直接应用多维度切片（小说文本没有序列标注）
2. LLM提取大纲时超时

### 解决方案

创建"小说转大纲驱动"，将完整小说转换为L2格式大纲：

```
小说文本 → 小说转大纲驱动 → L2大纲 → 标注数据 → 多维度切片 → 双库存储 → Agent检索
```

## 小说转大纲驱动设计

### 功能模块

1. **文本分块**：将长小说分割为适合LLM处理的块
2. **基本信息提取**：小说名、作者、类型、主角、金手指
3. **序列识别**：识别剧情序列，提取功能链、情绪曲线
4. **人物提取**：提取主要人物及行动元
5. **大纲生成**：输出L2格式的结构化大纲

### 输出格式

```json
{
  "novel_name": "1627崛起南海",
  "author": "零点浪漫",
  "genre": "历史穿越、工业建设",
  "protagonist": "肖林",
  "golden_finger": "穿越团队+现代知识",
  "core_appeal": "工业建设、强国崛起",
  "sequences": [
    {
      "name": "穿越开局",
      "functions": ["团队聚集", "穿越事件", "到达明朝"],
      "emotion_curve": "迷茫→决心→希望",
      "appeal_type": "开篇钩子",
      "characters": ["肖林", "穿越团队"]
    }
  ]
}
```

### 后续优化

1. **分块处理**：长文本分块，逐块提取后合并
2. **重试机制**：API超时时自动重试
3. **缓存中间结果**：避免重复提取
4. **批量处理**：支持多本小说批量转换
