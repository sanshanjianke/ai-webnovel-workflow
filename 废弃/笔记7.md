## 二、关键设计

### 1. 文档增强（核心创新）

**问题**：纯向量检索对中文专有名词效果差，"郑屠"搜索返回不相关结果

**解决方案**：多视角改写，增加检索入口

```
原始序列：李白制服郑屠

文档增强（5个视角）：
├─ 情节概述: "李白凭借大魔头气场震慑发狂患者郑屠"
├─ 爽点分析: "反差装逼与气场压制的完美结合"
├─ 人物关系: "李白与郑屠是压制与被压制关系"
├─ 情感弧线: "迷茫→紧张→惊讶→爽快"
└─ 读者问题: ["郑屠被李白的气场震慑产生幻觉？", ...]
```

**效果对比**：
| 查询 | 无增强 | 有增强 |
|------|--------|--------|
| "郑屠" | seq_015 (错误) | seq_001 (正确) |
| "武疯子被枪毙" | 无结果 | seq_021 (正确) |
| "戏弄道士" | 无结果 | seq_006 (正确) |

### 2. 多维度切片

同一内容按不同维度切分，适应不同检索场景：

| 维度 | 切片单位 | 边界 | 适用场景 |
|------|---------|------|---------|
| plot | 序列 | 序列开始/结束 | 剧情架构师查询完整情节 |
| character | 人物段落 | 人物出场/退场 | 人物设计师查询角色行为 |
| emotion | 情绪段落 | 情绪转折点 | 网络编辑评估爽点节奏 |
| function | 叙事功能 | 功能边界 | 分析剧情结构 |

**示例**：
```
序列"魔头苏醒与职场首秀"(5000字)

plot维度:
  └─ 1个切片: 完整序列

character维度:
  ├─ 李白切片: 李白戏份
  ├─ 清瑶切片: 清瑶戏份
  ├─ 郑屠切片: 郑屠戏份
  └─ 周真人切片: 周真人戏份

emotion维度:
  ├─ 迷茫阶段
  ├─ 惊讶阶段
  ├─ 从容阶段
  └─ 霸气阶段

function维度:
  ├─ 铺垫功能
  └─ 获得功能
```

### 3. 混合检索

**向量检索问题**：text-embedding-v3对中文专有名词处理不佳

**解决方案**：向量搜索 + 关键词过滤

```python
# 混合得分计算
vector_score = 1 - min(distance / 1.5, 1)  # 距离转0-1
keyword_score = matched_keywords / total_keywords  # 关键词匹配率
hybrid_score = vector_score * 0.6 + keyword_score * 0.4
```

### 4. 上下文记忆

Step5串行处理，保持上下文连贯：

```
Block 1 → 提取序列 → 输出outline_01.json
    ↓
    传递上下文摘要（已提取的人物+序列）
    ↓
Block 2 → 继续提取 → 输出outline_02.json
    ↓
    ...
```

上下文摘要包含：
- 已提取人物表（前10个）
- 已提取序列名（后5个）

## 四、使用方法

### 完整流程

```bash
cd /home/ssjk/talk/hybrid_rag_prototype

# Step 1: 章节切片
./venv/bin/python steps/step1_split_chapters.py \
    /home/ssjk/talk/都市剑说.txt \
    --output output/01_chapters \
    --max 300

# Step 2: 序列分组
./venv/bin/python steps/step2_create_sequences.py \
    --input output/01_chapters \
    --output output/02_sequences \
    --size 7

# Step 3: LLM生成草稿 (并行)
./venv/bin/python steps/step3_generate_drafts.py \
    --input output/02_sequences \
    --output output/03_drafts \
    --workers 10 \
    --timeout 180

# Step 4: 切分成块
./venv/bin/python steps/step4_merge_blocks.py \
    --input output/03_drafts \
    --output output/04_blocks \
    --size 50

# Step 5: LLM生成L2大纲 (串行)
./venv/bin/python steps/step5_generate_outline.py \
    --input output/04_blocks \
    --output output/05_outline \
    --name "都市剑说" \
    --timeout 180

# Step 6: 文档增强+切片
./venv/bin/python steps/step6_enrich_slices.py \
    --input output/05_outline \
    --output output/06_slices \
    --timeout 180

# Step 7: 向量存储
./venv/bin/python steps/step7_vector_store.py \
    --input output/06_slices \
    --output output/07_vector_db
```

### 检索使用

```python
from steps.retriever import create_retriever
from pathlib import Path

# 创建检索器
retriever = create_retriever(
    vector_db_path=Path('output/07_vector_db'),
    embedding_type='dashscope',
    api_key='your-api-key'
)

# 语义搜索
results = retriever.search('李白怎么制服郑屠', k=3)
for r in results:
    print(f"{r.name} (得分: {r.score:.3f})")

# 标签搜索
results = retriever.search_by_tags({'appeal_types': '打脸'}, k=5)

# 指定维度搜索
results = retriever.search('穿越归来', dimension='plot', k=3)
```

## 七、关键技术点

### 1. 断点续传

所有LLM调用步骤都支持断点续传：

```python
# Step 3: 检查已完成的草稿
if draft_file.exists():
    print("已存在，跳过")
    continue

# Step 5: 检查已完成的outline
if outline_file.exists():
    load_and_continue()

# Step 6: 检查进度文件
progress_file = output_dir / ".progress.json"
if progress_file.exists():
    enrichments = json.loads(progress_file.read_text())
```

### 2. 进度显示

Step 6 带进度条：

```
[█████████████████████████░░░░░] 24/28 政治博弈与强制休假 ✓
```

包含：进度条、当前/总数、序列名、状态、ETA

### 3. 超时重试

```python
for attempt in range(max_retries):
    try:
        response = llm.invoke(prompt)
        return parse_response(response)
    except Exception as e:
        if attempt < max_retries - 1:
            print(f"重试 {attempt+2}/{max_retries}...")
            time.sleep(2)
        else:
            print(f"失败: {e}")
```

### 4. 中文文本处理

```python
# 中文字符UTF-8编码约3字节
# 50KB ≈ 50000字节 ≈ 16000中文字符
block_size_chars = block_size_kb * 1024 // 3
```
