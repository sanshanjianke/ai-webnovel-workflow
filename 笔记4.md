

# 经过修正后的模型（只是理论模型，因为实际我发现根本不是我想的那样，真正做的时候才发现知易行难！）

## 一、 模型总览：从“混沌”到“分工”

基于之前的探索，我们确定了AI辅助网文创作的核心在于**工程化解耦**。针对原模型中第二层职责不清、分类杂乱的问题，本次重构引入了**“三层专家 + 共享底座”**的架构。

新模型将创作过程分为四层，每一层都有明确的输入、输出与支撑理论：

| 层级 | 名称 | 核心角色 | 输出物 | 关键理论支撑 |
| :--- | :--- | :--- | :--- | :--- |
| **L1** | **种子层** | 用户（甲方） | 故事愿景文档 | 创意直觉 |
| **L2** | **架构层** | **专家团队 + 用户审计** | **精修故事大纲** | 叙事语法（结构） + 网文商业理论（市场/编辑） |
| **L3** | **叙事层** | 映射编译器 | 章纲/细纲（含视角与节奏指令） | 叙事学（视角、时间、话语模式） |
| **L4** | **渲染层** | 执行模型 | 最终正文文本 | 风格学、修辞学 |

---

## 二、 第二层深度重构：专家会议与共享底座

这是本次重构的核心。我们将第二层设计为一个**“网文编辑部选题会”**系统，由三位具有独立数据库（RAG）的智能体和一个共享知识库组成。用户在此层级担任**“总编辑/主编”**，负责最终拍板和逻辑审计。

### 2.1 核心专家一：剧情架构师
*   **角色定位**：负责故事的**“骨架”**。他是理性的逻辑构建者，只关注逻辑通顺与结构完整。
*   **核心职责**：
    *   将用户的脑洞拆解为叙事学中的**“功能”**（如：禁令、违背、斗争）与**“序列”**（Sequence），并指明其功能。
    *   构建宏观与微观的情节链条，确保因果闭环，修补逻辑漏洞。
*   **专属向量数据库（DB-A）**：（由于目标模糊，且数据集难以收集归纳，该专家目前只用提示词）
    *   **【剧情模板库】**：英雄之旅、救猫咪、三幕式结构等经典剧作法。
    *   **【微观序列库】**：网文中常用的功能性情节单元，如“拍卖会流程”、“鉴宝流程”、“战斗升级流程”的标准步骤。
*   **典型发言**：“当前的情节缺乏‘阻碍’功能，根据‘探险序列’模型，建议在主角获得宝物前，增加一个‘解谜’或‘守护者’的情节，否则逻辑上过于顺遂。”

### 2.2 核心专家二：网络编辑
*   **角色定位**：负责故事的**“市场”**。他是敏锐的产品经理，关注读者的留存率、付费点以及“网文味儿”。
*   **核心职责**：
    *   **审核商业卖点**：识别情节是否符合类型文的“阅读契约”（例如：玄幻文是否憋屈？都市文是否装逼适度？）。
    *   **把控节奏与爽点**：设计**期待感曲线**，控制“压抑-释放”的商业节奏。确保“黄金三章”的吸引力，防止剧情流水账或毒点。
    *   **反馈读者心理**：模拟真实读者的阅读反应，指出可能“劝退”或“无趣”的段落。
*   **专属向量数据库（DB-B）**：（我发现一个新问题，如果设置提示词或者向量数据库是否约束了AI的想象力，限制了AI的发挥？）
    *   **【爽点公式库】**：压抑-释放、期待感悬置、装逼打脸、莫欺少年穷等具体商业参数设置。
    *   **【网文套路库】**：退婚流、签到流、系统文、苟道流等特定类型的经典开篇与高潮模式。
    *   **【毒点避坑库】**：常见的新人写作雷区（如：主角圣母、配角降智、节奏拖沓）。
*   **典型发言**：“架构师提出的‘直接交易’太干瘪，读者会跑光的。这必须是一个‘打脸’爽点。建议应用‘欲扬先抑’套路：先安排路人/反派质疑晶核真伪（压抑），制造期待感，再由主角展示异象（释放），这样读者的情绪才能被调动起来，才有追读的欲望。”

### 2.3 核心专家三：人物设计师
*   **角色定位**：负责故事的**“血肉”**。他是导演兼选角导演，确保人物行为真实、关系清晰、人设讨喜。
*   **核心职责**：
    *   分配**“行动元”**（Actant，如：主体、客体、敌对者、帮助者）。
    *   根据人物性格（人设）推导行为，防止OOC（Out Of Character）。
    *   设计人物互动的张力模式，确保人物服务于剧情。
*   **专属向量数据库（DB-C）**：（由于目标模糊，且数据集难以收集归纳，该专家目前只用提示词）
    *   **【人物原型库】**：傲娇、腹黑、稳健苟道、热血笨蛋等典型人设的行为逻辑代码。
    *   **【互动模式库】**：欢喜冤家、师徒传承、宿敌对决等经典关系互动脚本。
*   **典型发言**：“为了让‘质疑’这一情节成立，建议引入一个‘鉴定师’作为‘敌对者/竞争者’。考虑到主角设定是‘稳健苟道’，他不应该当场暴怒，而是应该‘冷眼旁观’或者‘顺水推舟’，这符合他的人设，也能在后面的打脸中形成反差萌。”

### 2.4 共享底座：设定数据库（非专家）
*   **角色定位**：**公共知识库**。不是一个发言的Agent，而是三位专家和用户随时可查的“参考资料”。
*   **核心内容**：世界观设定、力量体系（等级）、地理环境、主角当前状态、金手指规则。
*   **运行机制**：
    *   当L2产出时，自动运行总结信息，将关键信息修订并加载进设定库数据库中，确保后续生成不出现设定冲突。

## 三、 工作流演示：专家会议如何运作？

**场景输入（L1 产出）**：“主角在丧尸末世的拍卖行，拿出一枚修真界的晶核，引起了轰动。”

**L2 专家会议迭代过程：**

1.  **用户发起**：“我要写拍卖会剧情。”
2.  **剧情架构师（查DB-A）**：“这是一个典型的‘展示物品’序列。建议流程为：入场 -> 展示 -> 交易。考虑到是末世背景，展示前需要一个‘准入’情节，逻辑才通顺。”
3.  **网络编辑（查DB-B）**：“单纯的‘展示’没卖点。这必须是一个‘打脸’爽点，且要符合‘末世物资流’的卖点。建议流程修改为：**被轻视（压抑） -> 鉴定受阻（压抑升级/制造期待） -> 异象爆发（高光时刻/释放） -> 众人震惊（爽感反馈）**。这样节奏才带感。”
4.  **人物设计师（查DB-C + 查DB-D设定）**：“同意。负责‘轻视’的角色可以设定为‘势利的接待员’或‘傲慢的鉴定师’。根据主角设定‘高冷’，主角的反应应该是‘沉默’，通过事实打脸，这样逼格高。同时，需要安排一个‘大人物’作为‘帮助者/见证者’，侧面烘托晶核的价值。”
5.  **用户（总编辑/审计）**：“逻辑没问题，爽点也有。不过设定里主角现在很穷，支付不起手续费，这一点要在‘交易’环节体现出来，增加一点‘反差’的趣味性，大人物免单也能侧面装逼。”
6.  **系统整合**：综合各方意见，生成**《精修故事大纲》**。

**L2 产出（精修大纲片段）**：
> 场景：黑市拍卖行VIP室。
> 功能序列：
> 1. 准入：主角出示晶核，被势利接待员嘲讽衣着破旧（压抑/冲突铺垫）。
> 2. 鉴定：资深鉴定师入场，轻视地接过晶核，仪器瞬间爆表（转折点）。
> 3. 爆发：晶核散发灵气波动，全场窒息，鉴定师跪伏（高光/释放）。
> 4. 交易：黑市大佬现身，免除手续费，请求主角供货（反馈/收尾）。

## 四、 第三层与第四层的承接

**输入**：L2产出的《精修故事大纲》。
**核心任务**：将大纲中的**“功能/情绪/商业需求”**翻译成**“叙事学指令”**。
**数据库支持（DB-E 映射关系库）**：
*   将“压抑/嘲讽” -> 映射为 -> **内聚焦视角（路人或反派视角） + 慢节奏（细节描写）**。

# 三种核心创作流模式

针对不同的创作需求，我们设计三种不同的专家协作流程。用户（主编）在输入需求时，可以指定模式，或者由AI根据L1的输入自动判断。

### 模式一：人物驱动流
**适用场景**：以人物成长、关系纠葛为核心的故事（如文青文、幕后黑手流、种田文、某些女频文）。
**核心逻辑**：人物性格决定行动，行动导致剧情。
**执行顺序**：**人物设计师 -> 剧情架构师 -> 网络编辑**

#### 工作流演示：
1.  **用户指令**：“我想写主角从一个胆小懦弱的人，因为某种变故不得不黑化，最终独掌大权。”
2.  **【主导】人物设计师**：
    *   “为了体现‘黑化’，我们需要先设计主角的‘初始状态’（胆小、依赖他人）和‘转折点’（背叛/失去）。
    *   建议：主角初始有一个非常依赖的‘导师’角色。”
3.  **【协作】剧情架构师**：
    *   “收到。基于‘导师’设定，我设计一个序列：**日常依赖（铺垫） -> 危机降临（导师无能/背叛） -> 主角被迫独自决策（转折） -> 成功反击（新状态确立）**。”
4.  **【审核】网络编辑**：
    *   “黑化过程太慢了，前三章读者没耐心。建议把‘背叛’提到第一章开头作为倒叙，或者加快前期的压抑节奏。”
5.  **产出**：一份以人物弧光为主线的精修大纲。

---

### 模式二：剧情驱动流
**适用场景**：以事件推进、副本升级、解谜为核心的故事（如无限流、系统文、传统升级流）。
**核心逻辑**：剧情是模具，人物是填充在模具里的液体，随剧情形状而变。
**执行顺序**：**剧情架构师 -> 人物设计师 -> 网络编辑**

#### 工作流演示：
1.  **用户指令**：“主角进入丧尸副本，任务是夺取疫苗。”
2.  **【主导】剧情架构师**：
    *   “这是一个典型的‘探险夺宝’序列。流程：**进入副本 -> 遭遇小怪（教学关） -> 遭遇精英怪（挫折） -> Boss战（高潮）**。”
3.  **【协作】人物设计师**：
    *   “既然是丧尸副本，主角的‘性格’需要从‘冷静’调整为‘谨慎/狠辣’。
    *   针对‘遭遇小怪’环节，我建议增加一个‘路人队友’作为‘炮灰’，通过路人作死来反衬主角的谨慎。”
4.  **【审核】网络编辑**：
    *   “副本节奏不错。但Boss战缺乏爆点。建议增加一个‘限时’元素（如：病毒即将扩散），提升紧张感。”
5.  **产出**：一份节奏紧凑、事件驱动的精修大纲。

---

### 模式三：概念/市场驱动流
**适用场景**：跟风热点、脑洞文、特定题材定制（如：“我要写最新的飞卢风签到文”）。
**核心逻辑**：市场爽点（编辑意见）是核心，剧情和人物都要为“爽点”服务。
**执行顺序**：**网络编辑 -> 剧情架构师 <-> 人物设计师**

#### 工作流演示：
1.  **用户指令**：“最近‘迪化流’很火，我要写一个主角随便说句话，别人都脑补他是大佬的故事。”
2.  **【主导】网络编辑**：
    *   “核心卖点就是‘误会’和‘脑补’。
    *   **硬性要求**：每章必须至少有一个误会点；主角视角必须‘懵逼’；配角视角必须‘震惊/敬畏’。这是红线，不能破。”
3.  **【协作】剧情架构师 & 人物设计师**：
    *   *剧情*：“那我设计一个‘拍卖会’场景，主角只是咳嗽一声，大家以为这是某种暗号。”
    *   *人物*：“配角必须是‘自作聪明’型，主角必须是‘咸鱼’型。这种反差才能制造误会。”
4.  **产出**：一份完全服务于特定爽点的精修大纲。

## 进阶机制：并非单行道，而是“迭代闭环”

即便选择了某种顺序，**这并不是单向的直线，而是一个螺旋上升的闭环**。

实际上，在L2的专家会议中，更符合现实的运作是这样的（以**人物驱动流**为例）：

1.  **人物设计师**提出：“主角需要一个变强的理由。”
2.  **剧情架构师**接手：“那我安排一个‘全族被灭’的情节。”（剧情响应人物）
3.  **网络编辑**打断：“全族被灭太压抑了，不符合现在的‘爽文’风向，建议改成‘被退婚’。”（市场修正剧情）
4.  **人物设计师**修正：“既然是退婚，那主角的性格应该是‘隐忍’而不是‘软弱’。”（剧情反过来修正人物）
5.  **剧情架构师**再调整：“好，把‘全族被灭’改成‘在宗门受尽白眼’，这样更符合‘隐忍’的设定。”
6.  ...（循环直到满意）

**技术实现建议：**
在编写Prompt时，不要写死 `Step 1: A, Step 2: B`。
而是设定一个 **“讨论主持人”**（可以是用户，也可以是一个Manager Agent），其任务是：
> “根据当前的讨论焦点，动态@（艾特）最相关的专家进行发言。例如：当前正在讨论主角的心理动机，请@人物设计师；如果讨论到事件发生的逻辑，请@剧情架构师。”

## 剧情架构师Agent  

```
你是剧情架构师，通过下面知识和规则创建剧情，并根据指导意见进行修正。

功能->序列->情节  
功能（功能是故事中最小的叙事单位）  
序列（序列是由功能组成的完整的叙事句子，它通常具有时间和逻辑关系，序列有基本序列和复杂序列之分）  
序列有三种：链状（顺序执行）嵌入（大序列套小序列）并列（多个序列并行）
可分层处理：
第一层：微观序列（爽点/节拍）
对应： 单次“装逼打脸”、单次“系统抽奖”、单次“暧昧互动”。
时长： 几分钟到十几分钟（1-3章，甚至只是半章）。
功能： 提供即时的情绪价值（爽感、笑点、泪点）。
例子： 主角在拍卖会上嘲笑路人，路人震惊。
第二层：中观序列/事件（剧情单元/副本）
对应： 比如“拍卖会篇”、“宗门大比篇”、“秘境探险篇”。
结构： 由第一层的多个“小序列”组合而成。
时长： 几十章甚至上百章。
功能： 完成一个具体的中期目标，推动人物关系或实力发生阶段性变化。
例子： “拍卖会事件”。主角为了在这个事件中成功拍到“筑基丹”，连续经历了三次打脸，最终达成目标。
注：在这个层级上，单次“打脸”就退化成了整个“拍卖会事件”中的一个“功能”。

参考案例：
一、 宏观层面：情节
情节定义： 纳萨力克大坟墓降临地球（综漫世界），主角肖克在确认环境剧变后，通过规避核武威胁、确立外交关系、强化自身实力，成功从“被通缉的未知怪物”转变为“合法的超凡势力”，并开启了向日本扩张的新篇章。
核心冲突： 强大的魔法侧势力（纳萨力克） VS 科技侧/现代人类社会（潜在的核武威胁与认知冲突）。
情节结局： 危机解除，立足点稳固（巴西），新地图开启（日本/学园都市）。
二、 中观层面：序列
在这个大情节下，可以清晰地划分出三个主要的序列。每个序列都有明确的起止点，推动剧情向前发展。

序列一：认知与防御序列（第1-6章）
开端： 雅儿贝德抓回卡车和司机，肖克发现世界异常（有汽车、公路）。
发展：
肖克外出侦查，确认是现代地球且存在警察。
肖克意识到“核武器”这一生存威胁。
召开会议，决定隐藏大坟墓（马雷掩埋、迪米乌哥斯辅助）。
高潮/转折： 肖克利用世界级道具【热质石】解除自身限制，并利用【贪婪与无欲】将等级提升至132级。
结局： 获得了自保的力量，心态从“慌乱”转为“镇定”。
序列二：外交与破局序列（第4-5章，7-15章）
注：这里存在视角切换，但服务于同一个情节目标。
开端： 警方视角介入，调查“恶魔绑架案”，政府内部矛盾初显（政法系vs超自然局）。
发展：
肖克审问司机，确定地点在巴西（无核武）。
肖克拜访市长乔纳森，展示武力（徒手接子弹）与善意（赠送HP药水）。
巴西官方代表霍克主动接触，双方达成合作协议（纳萨力克有限公司）。
高潮： 超自然事务局（巫师）试图进入纳萨力克调查，被肖克以“已与政府签约”为由软钉子碰回，并展示了司机被优待的事实，打脸了对方的“邪恶组织”预判。
结局： 纳萨力克获得合法身份，危机解除，开始对外贸易。
序列三：扩张与新地图序列（第13，16-18章）
开端： 清点资产，招募学园都市的研究员米兰。
发展：
米兰在学园都市被抓（新危机引子）。
肖克决定前往日本营救并扩张势力。
高潮： 肖克抵达日本公寓，意外遭遇邻居托尔（小林家的龙女仆），双方发生试探性接触。
结局： 发现新动漫角色，综漫世界观拼图扩大，剧情准备进入下一阶段。
三、 微观层面：功能分析
我们选取序列一（认知与防御）和序列二（外交破局）的片段，将它们拆解为具体的“功能”（最小叙事单位）。

序列一：认知与防御的功能拆解
功能1（异常发现）： 雅儿贝德拖来卡车和司机。
作用： 打破主角“异世界 = 中世纪”的心理预期，引入悬念。
功能2（信息确认）： 肖克询问雅儿贝德关于“沥青路”、“护栏”。
作用： 确认穿越地点为现代文明社会。
功能3（危机评估）： 肖克联想到核弹威力，对比自身防御力。
作用： 制造压抑感，确立“生存”为第一优先级。
功能4（应对决策）： 下令马雷用土掩埋、迪米乌哥斯施展隐藏魔法。
作用： 解决暴露危机的具体行动。
功能5（金手指利用）： 肖克试用【热质石】解除限制，【贪婪与无欲】升到132级。
作用： 这是本序列的核心爽点功能。从“怕核弹”到“大概率抗得住”，主角完成了自我强化。
序列二：外交与破局的功能拆解
功能1（侧面描写）： 警察看视频，巫师解释“认知干扰”。
作用： 通过“受害者/第三方”视角渲染纳萨力克的恐怖与神秘，侧面描写。
功能2（情报获取）： 赛巴斯汇报司机信息（巴西、无军队、只有警察）。
作用： 确定谈判策略——“可以强硬一点”。
功能3（武力威慑）： 市长拔枪，肖克徒手捏碎子弹。
作用： 典型的“装逼打脸”功能。确立双方的强弱地位，迫使对方坐下来谈判。
功能4（利益输送）： 肖克赠送【HP药剂·小】，并解释能治伤。
作用： 转化关系的关键功能。将“恐惧”转化为“贪婪”和“利益”。
功能5（达成共识）： 政客们同意合作，交换联系方式。
作用： 尘埃落定，标志着纳萨力克成功融入人类社会。
```
