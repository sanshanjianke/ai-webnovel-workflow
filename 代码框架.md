### 2.1 启动流程

```
main.py lifespan
    │
    ├── load_config("data/user/config.yaml")
    │   └── 如果文件不存在 → 创建默认配置
    │   └── 返回 AppConfig（Pydantic 模型）

        ├── 遍历 backend/modules/{llm,rag,worldbook,l1,l3,l4}/
        │   └── import 每个 .py 文件
        │   └── @register_module 装饰的类自动进入 MODULE_REGISTRY
        ├── import backend/modules/experts
        ├── import backend/modules/orchestration
        ├── 遍历 backend/meeting_protocols/*.py
        ├── 兼容旧路径 import backend/modules/l1_5
        └── 加载用户自定义专家（data/user/custom_experts/）
```

### 2.2 插件注册机制

```python
# backend/core/registry.py 核心逻辑

MODULE_REGISTRY: dict[str, dict[str, type]] = {
    "llm": {}, "rag": {}, "worldbook": {},
    "l1": {}, "l3": {}, "l4": {},
    "expert": {}, "meeting_protocol": {},
}

def register_module(category):
    """装饰器工厂：将类注册到指定类别"""
    def decorator(cls):
        # 类名 CamelCase → snake_case
        key = re.sub(r'(?<!^)(?=[A-Z])', '_', cls.__name__).lower()
        # 去除已知后缀（_provider, _agent, _generator 等）
        for suffix in ['_provider', '_agent', '_generator', '_architect',
                       '_designer', '_editor', '_author', '_representative',
                       '_compiler', '_renderer', '_retriever',
                       '_splitter', '_summarizer']:
            if key.endswith(suffix):
                key = key[:-len(suffix)]
                break
        MODULE_REGISTRY[category][key] = cls
        return cls
    return decorator
```

**使用示例**：

```python
# backend/modules/llm/openai_compat.py
from backend.core.registry import register_module
from backend.core.protocols.llm import BaseLLMProvider

@register_module("llm")
class OpenAICompat(BaseLLMProvider):
    ...

# 注册后可通过 get_module("llm", "open_ai_compat") 获取此类
```


**会议引擎的调用流程**（`backend/modules/orchestration/__init__.py`）：

```python
engine = MeetingEngine(config)
# config 是 MeetingConfig，包含：
#   - experts: 专家节点列表（每位专家的 expert_id, role, custom_prompt）
#   - containers: 容器节点列表（内部包含的子节点和边）

    yield SSE(event)

for event in engine.run_pipeline(context_input, worldbook_text, rag_context, human_feedback):
    yield SSE(event)
```

```python
class ConstrainedRenderer(BaseL4Renderer):
    def stream_render(self, chapter_plan, rag_technique, world_book):
        yield f"# {chapter_plan.chapter_name}\n\n"
        for scene in chapter_plan.scenes:
            constraints = self._build_constraints(scene)  # 视角/节奏约束
            requirements = self._build_scene_requirements(scene)  # 场景内容

                yield chunk
```

### 4.2 SSE 通信模式

**模式一：POST 流式（L1Seed、Orchestration）**

```javascript
// fetch + ReadableStream，手动解析 SSE 格式
const response = await fetch(url, { method: 'POST', body: JSON.stringify(data) })
const reader = response.body.getReader()
const decoder = new TextDecoder()

while (true) {
  const { done, value } = await reader.read()
  if (done) break
  buffer += decoder.decode(value, { stream: true })
  // 按 \n 分割行，检测 "data:" 前缀，JSON.parse
  // 根据 data.type 分发事件
}
```

**模式二：GET 流式（L4Render、旧 L15Meeting、旧 L2Meeting）**

```javascript
// 原生 EventSource
const es = new EventSource(`/api/projects/${id}/l4/stream`)
es.addEventListener('text', (e) => { /* 追加文本块 */ })
es.addEventListener('done', () => { es.close() })
```

### 4.3 编排画布（`OrchestrationCanvas.vue`）

基于 Vue Flow（`@vue-flow/core`）实现。关键逻辑：

- **工具箱**：左侧列出所有已注册专家（builtin + 自定义），可拖拽到画布
- **节点**：`ExpertNode.vue`（专家）和 `GroupNode.vue`（容器），通过 Vue Flow 的 template node 机制渲染
- **连线**：`speak_order`（实线箭头）、`query`/`update`（虚线）
- **拓扑排序**：`orderedNodes` computed 属性对 DAG 做 Kahn 算法，确定运行时执行顺序
- **预设模板**：点击预设卡片 → 自动加载对应的节点布局
- **保存/加载设计**：画布布局 JSON 化存入文档库（`layer: "orchestration"`），可从侧边栏拖回加载

### 4.4 文档库侧边栏（`library/DocumentSidebar.vue`）

- **布局**：可折叠（40px 收起 / 280px 展开 / 200-600px 拖拽调整）
- **目录树**：递归渲染，支持展开/折叠/右键菜单
- **拖拽**：
  - 内部：拖到目录 → 移动文档
  - 外部：拖出到其他页面 → 传递文档 UID 和内容，其他页面可接收
- **活跃文档指示**：绿色圆点标记各层当前活跃文档
- **上下文菜单**：重命名、归档、导出、删除、设为活跃

## 五、服务层（services/）

所有 service 都是单例模式，通过 `get_xxx()` 函数获取实例：

```python
# project_manager.py
project_manager = get_project_manager()
project = project_manager.create_project("我的小说", ...)
projects = project_manager.list_projects()

# meeting_logger.py
logger = get_meeting_logger(project_id)
logger.log_meeting(project_id, expert_id, expert_type, content, suggestions, round_num)

# library_manager.py
library = get_library_manager(project_path)
uid = library.add_document(entry, content)
document = library.get_document(uid)

# generation_manager.py
gen_mgr = get_generation_manager()
gen_mgr.start_generation(project_id)
# ... 在生成循环中检查 ...
if gen_mgr.should_stop(project_id):
    break
gen_mgr.end_generation(project_id)
```

各服务的数据存储：

| 服务 | 存储 | 格式 |
|------|------|------|
| ProjectManager | `data/projects/{id}/project.json` | JSON |
| MeetingLogger | `data/projects/{id}/logs/meetings.db` | SQLite（3 表：meeting_logs, version_history, rag_logs） |
| LibraryManager | `data/projects/{id}/library/manifest.json` + `files/{uid}.json` | JSON |
| WorldBookManager | 委托给 WorldBook 实例的方法 | JSON |
