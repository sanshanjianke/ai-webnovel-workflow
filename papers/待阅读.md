## 一、多智能体系统 (Multi-Agent Systems)

### 1. Agents' Room: Multi-Step Collaboration for Story Generation ⭐⭐⭐⭐⭐

**文件:** `Agents_Room_Multi-Step_Collaboration_for_Story_Generation.pdf`  
**作者:** Google DeepMind (Fantine Huot et al.)  
**发表:** ICLR 2025  
**arXiv:** 2410.02603

---

#### 摘要 (Abstract)

写作引人入胜的小说是一个多层面的过程，需要结合构建情节、塑造有趣角色、使用富有表现力的语言等元素。虽然大语言模型(LLM)在故事写作方面展现出潜力，但目前主要依赖复杂的提示工程，这限制了其实际应用。

本文提出 **AGENTS' ROOM**，一个受叙事理论启发的生成框架，将叙事写作分解为由专业智能体处理的子任务。该方法包含：
- **规划智能体(Planning Agents):** 负责故事关键组件（角色、情节、冲突等）但不直接写作
- **写作智能体(Writing Agents):** 负责生成最终文本，按章节专业化分工
- **协调器(Orchestrator):** 管理智能体调用顺序
- **草稿板(Scratchpad):** 共享信息的管理机制

同时贡献了 **TELL ME A STORY** 数据集，包含复杂写作提示和人类撰写的故事，以及专门用于评估长篇叙事的新评估框架。

实验结果表明，AGENTS' ROOM生成的故事在人类专家评估和自动指标上均优于基线系统。

---

#### 研究问题

1. **核心问题:** 如何将复杂的长篇故事写作任务分解为可管理的子任务？

2. **LLM写作的挑战:**
   - 维持长文本的一致性、语调和事实准确性
   - 缺乏独特的声音或幽默感
   - 往往复制训练数据中的模式，缺乏原创性
   - 缺乏长篇写作数据集和标准化评估标准

3. **现有方法局限:**
   - 依赖详细的提示工程
   - 提示链(Prompt Chaining)复杂且难以控制
   - 单一智能体难以处理多维度写作任务

---

#### 核心方法

**1. 多智能体架构**

```
┌─────────────────────────────────────────┐
│           Orchestrator (协调器)          │
│        管理智能体调用顺序                │
└─────────────────────────────────────────┘
                    │
        ┌──────────┴──────────┐
        ▼                     ▼
┌──────────────┐      ┌──────────────┐
│  Planning    │      │   Writing    │
│   Agents     │      │    Agents    │
│  (规划智能体) │      │  (写作智能体) │
└──────────────┘      └──────────────┘
        │                     │
        └──────────┬──────────┘
                   ▼
        ┌──────────────────┐
        │   Scratchpad     │
        │    (草稿板)       │
        │ 信息共享与维护    │
        └──────────────────┘
```

**2. 智能体分工**

| 类型 | 职责 | 示例 |
|------|------|------|
| **Planning Agents** | 规划故事组件 | 角色描述、情节设计、冲突设置、世界观构建 |
| **Writing Agents** | 生成最终文本 | 引言、上升动作、高潮、下降动作、结局 |
| **Orchestrator** | 协调执行顺序 | 按叙事理论调度各智能体 |

**3. 叙事理论启发**

基于经典叙事结构：
- ** exposition ( exposition )**
- **Rising Action (上升动作)**
- **Climax (高潮)**
- **Falling Action (下降动作)**
- **Resolution (结局)**

每个阶段由专门的智能体负责。

**4. 多智能体优势**

- ✅ LLM可专业化为零样本提示或微调的单一功能智能体
- ✅ 避免长而模糊的指令需要多轮迭代的问题
- ✅ 适用于解决方案未知、需要探索的研究空间
- ✅ 天然支持人在回路(Human-in-the-loop)

---

#### 主要贡献

1. **AGENTS' ROOM框架**
   - 首个将叙事理论与多智能体协作结合的写作系统
   - 模块化设计，支持不同叙事结构

2. **TELL ME A STORY数据集**
   - 高质量人类写作提示和故事
   - 复杂写作场景

3. **长篇叙事评估框架**
   - 多维度故事质量评估
   - 人类评估 + 自动指标

4. **实验验证**
   - 专家评估优于基线系统
   - 广泛的自动化和人类指标分析

---

#### 实验结果

**1. 人类评估偏好**
- AGENTS' ROOM生成的故事被专家评估者更偏好
- 优于无协作/无专业化的基线系统

**2. 评估维度**
- 情节连贯性
- 角色一致性
- 语言质量
- 整体吸引力

**3. 故事长度**
- 生成1,000-2,000 tokens的长篇故事
- 保持跨段落一致性

**核心启示:**
- ✅ **验证了L2三专家会议架构的正确性** - 专业分工+协作=更好结果

#### 个人总结

**💡 关键启发:**

### 2. AgentVerse: Facilitating Multi-Agent Collaboration ⭐⭐⭐⭐

**文件:** `AgentVerse_Facilitating_Multi-Agent_Collaboration.pdf`  
**作者:** Tsinghua University (Weize Chen et al.)  
**发表:** arXiv 2023  
**arXiv:** 2308.10848

---

#### 摘要

受人类群体协作启发，本文提出多智能体框架 **AGENT VERSE**，能够有效编排专家智能体协作组，实现"整体大于部分之和"的系统效果。

该框架将问题解决过程模拟为人类群体的问题解决流程，分为四个关键阶段：
1. **Expert Recruitment (专家招募):** 根据当前进展动态调整智能体组成员
2. **Collaborative Decision-Making (协作决策):** 选定智能体联合讨论制定策略
3. **Action Execution (行动执行):** 智能体与环境交互实施行动
4. **Evaluation (评估):** 评估当前状态与目标差异，不满意则反馈迭代

实验覆盖文本理解、推理、编程、工具使用和具身AI等领域，证明多智能体组优于单一智能体。分析还发现了智能体协作中出现的涌现行为：志愿行为、从众行为、破坏行为。

---

#### 研究问题

1. **核心问题:** 如何设计一个通用的多智能体协作框架，使智能体组性能超越单一智能体？

2. **背景挑战:**
   - 复杂现实任务需要个体协作才能更好完成
   - 现有研究主要关注单一智能体能力
   - 缺乏系统化的多智能体协作框架

3. **人类协作启发:**
   - 人类团队协作可提升工作效率和效果
   - 智能体社会/群体概念兴起
   - 需要模拟人类群体的问题解决流程

---

#### 核心方法

**1. AGENT VERSE四阶段框架**

```
┌─────────────────────────────────────────────────────────┐
│  Iteration x N rounds (迭代N轮)                         │
└─────────────────────────────────────────────────────────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    ▼                      ▼                      ▼
┌──────────┐      ┌─────────────────┐     ┌──────────┐
│ Expert   │      │ Collaborative   │     │ Action   │
│Recruit-  │ ───▶ │ Decision-Making │ ──▶ │Execution │
│  ment    │      │   (协作决策)     │     │ (执行)   │
└──────────┘      └─────────────────┘     └──────────┘
     ▲                                              │
     │                                              ▼
     │                                       ┌──────────┐
     │                                       │   Group  │
     └───────────────────────────────────────│  Reward/ │
            Feedback (反馈)                  │ Feedback │
                                            └──────────┘
```

**2. 四阶段详解**

| 阶段 | 功能 | 说明 |
|------|------|------|
| **Expert Recruitment** | 专家招募 | 动态调整智能体组成，根据任务需求招募不同专家 |
| **Collaborative Decision-Making** | 协作决策 | 智能体联合讨论，制定问题解决策略 |
| **Action Execution** | 行动执行 | 智能体与环境交互，执行决策 |
| **Evaluation** | 评估反馈 | 对比当前状态与目标，不满意则进入下一轮迭代 |

**3. MDP建模**

整个过程建模为马尔可夫决策过程 (S, A, T, R, G):
- **S:** 智能体和环境状态空间
- **A:** 行动空间
- **T:** 状态转移
- **R:** 奖励函数
- **G:** 目标

**4. 涌现行为发现**

| 行为类型 | 描述 | 影响 |
|---------|------|------|
| **Volunteer (志愿)** | 智能体主动协助同伴 | ✅ 提升团队效率 |
| **Conformity (从众)** | 智能体调整偏离行为以符合共同目标 | ✅ 增强协作 |
| **Destructive (破坏)** | 偶尔出现的不期望行为 | ❌ 可能导致负面结果 |

---

#### 主要贡献

1. **AGENT VERSE框架**

   第二轮: 编辑专家审核 → 修改节奏/爽点

### 1. NarraBench: Narrative Benchmarking Framework ⭐⭐⭐⭐⭐

**文件:** `NarraBench_Narrative_Benchmarking_Framework.pdf`  
**作者:** Cornell University + McGill University  
**arXiv:** 2510.09869

#### 一句话总结
首个系统化的叙事理解评估框架，调查发现仅27%叙事任务被现有基准覆盖，提出12维叙事特征分类法。

#### 核心发现
- **覆盖缺口:** 仅27%叙事理解任务被现有基准充分覆盖
- **缺失领域:** 叙事事件、风格、视角、揭示等几乎未被评估
- **理论框架:** 12维叙事特征分类法(Big-4: story/narration/discourse/situatedness)

#### 12维叙事特征
1. Story(故事): 情节、事件、角色
2. Narration(叙述): 视角、声音、时间
3. Discourse(话语): 结构、风格、连贯性
4. Situatedness(情境): 文化背景、读者反应

#### 关键洞见
- 叙事理解是**解释性**任务(主观)，非**确定性**任务(客观)
- 现有基准过度关注故事内容，忽略话语结构和叙事视角
- 需要评估主观性和视角性特征(无唯一正确答案)

#### 与本项目关联 ⭐⭐⭐⭐⭐
- ✅ **L1/L2层评估体系基础** - 建立网文叙事评估的理论框架
- ✅ **12维特征** - 可直接用于本项目的故事特征提取
- ✅ **主观性评估** - 与100-Endings指标思路一致

#### 关键启示
- 本项目应建立网文专属的12维评估体系
- 爽点/毒点/节奏可纳入"Situatedness"维度
- 需要设计主观性评估方法(多人标注+一致性)

- **Press(环境):** 创作情境

- 网文AI应追求**组合创新**而非**原创突破**

### L1种子层 (创意生成)

**核心洞察:**
- **Boden三标准**: Value(价值) + Novelty(新颖) + Surprise(惊喜)
- **CreativityPrism三维**: Quality + Novelty + Diversity
- **人类创造力**: 组合创新为主，原创突破为辅

**应用技术:**
- Few-shot prompting快速原型
- APE自动提示优化
- 层次化生成(Premise→Title→Setting→Story)

---

### L1.5卷纲层 (整体规划)

**核心洞察:**
- **AgentWrite**: 分阶段长文本生成(规划→起草)
- **Recursive Reprompting(Re3)**: 计划→起草→重写→编辑

**应用技术:**
- 骨架-填充(Skeleton-of-Thought)
- 大纲控制(DOC)

---

### L2架构层 (情节架构)

**核心洞察:**
- **三专家分工**: 架构师(功能序列) + 人物师(行动元) + 编辑(爽点/毒点)
- **多智能体协作**: AGENTS' ROOM(专业化) + AgentVerse(四阶段迭代) + MetaGPT(SOP)
- **迭代优化**: 生成→评审→反思→改进

**应用技术:**
- AutoGen框架
- CAMEL角色扮演
- Tree of Thoughts多路径探索
- LATS(MCTS规划)
- Reflexion自我反思

**评估工具:**
- CHIRON角色表表示
- NarraBench 12维度

---

### L3叙事层 (章纲→细纲)

**核心洞察:**
- **反思驱动**: SuperWriter(规划→写作→精炼)
- **推理增强**: Chain-of-Thought, ReAct, RAP

**应用技术:**
- CoT逐步推理
- ReAct(推理+行动交替)
- 工具调用(Toolformer)

---

### L4渲染层 (正文生成)

**核心洞察:**
- **分阶段生成**: LongWriter(Plan→Draft→LongForm)
- **人机协作**: Wordcraft编辑器
- **长文本记忆**: RecurrentGPT(短期+长期记忆)

**应用技术:**
- LongWriter-AgentWrite
- DOC大纲控制
- Self-RAG按需检索
- Skeleton-of-Thought并行生成

---

### 世界书与RAG

**核心洞察:**
- **知识表示**: 传统KG + 向量DB混合
- **一致性**: CHIRON验证 + 冲突检测
- **检索**: RAG + Self-RAG

**应用技术:**
- 向量数据库(Chroma)
- 实体链接/指代消解
- ATOMIC/COMET常识推理
