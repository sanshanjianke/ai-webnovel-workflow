# 开发计划

> 原型开发阶段，每阶段产出可运行的 MVP。后续迭代中重构。

## 阶段一：核心基础设施

**目标**：项目骨架能跑，配置能加载，LLM 能调用。

- [ ] 项目初始化：`start.sh`/`start.bat`、`requirements.txt`、虚拟环境
- [ ] FastAPI 应用骨架：`backend/main.py`、CORS、基础路由
- [ ] 配置系统：加载 `data/user/config.yaml`，运行时读取
- [ ] `BaseLLMProvider` 接口 + OpenAI 兼容实现
- [ ] 模块注册表：扫描 `backend/modules/` 和 `meeting_protocols/`，发现 ABC 子类
- [ ] 基础测试框架搭建

**验证**：`./start.sh` 启动后访问 `http://localhost:7860` 返回 200。

---

## 阶段二：数据层

**目标**：项目创建、世界书读写、日志记录。

- [ ] 项目管理：`project.json` 读写，创建/列表/配置更新
- [ ] 世界书基础：`BaseWorldBook` 接口 + ST 风格实现（JSON + 触发词 + 读写）
- [ ] 版本控制：`commit()`/`revert()` 基础实现
- [ ] SQLite 日志：meeting.db 建表，会议记录写入/查询
- [ ] `data/` 目录结构初始化
- [ ] 项目 API：`POST /api/projects`、`GET /api/projects/{id}`、`PUT config`

**验证**：通过 API 创建项目，写入世界书条目，`commit()` 保存，`revert()` 回滚。

---

## 阶段三：L1 种子层

**目标**：用户输入创意，产出故事愿景。

- [ ] `BaseSeedGenerator` 接口
- [ ] `GuidedFormGenerator` 实现：引导式表单 → 填充模板输出 VisionDocument
- [ ] L1 API：`POST /api/projects/{id}/l1/generate`
- [ ] 前端 L1Seed 页面：表单 + AI 辅助展开 + 可编辑产出

**验证**：填写核心梗/阅读契约/粗略大纲 → 产出 VisionDocument JSON。

---

## 阶段四：L2 专家会议（核心）

**目标**：三专家群聊式讨论，产出精修大纲。

- [ ] `BaseExpert` 接口
- [ ] `PlotArchitect` v1（Propp 功能）实现
- [ ] `WebEditor` 实现
- [ ] `CharacterDesigner` 实现
- [ ] `BaseMeetingProtocol` 接口
- [ ] `plot_driven` 协议实现
- [ ] `character_driven` 协议实现
- [ ] `market_driven` 协议实现
- [ ] `ExpertMeetingL2` 编排器：按协议调用专家、处理迭代
- [ ] `human_feedback` 回调机制：`waiting_user` SSE 事件 + 前端决策按钮
- [ ] L2 SSE 流式 API：`GET /api/projects/{id}/l2/stream`
- [ ] L2 操作 API：`POST /l2/message`、`/l2/decide`、`/l2/continue`
- [ ] 前端 L2Meeting 页面：群聊 UI（角色颜色+头像、发言卡片、用户输入、决策按钮、右侧大纲预览）
- [ ] 协作模式切换（semi_auto/full_auto/manual）
- [ ] 错误处理：超时重试、显示错误卡片

**验证**：L1 产出输入 L2 → 三位专家按顺序发言 → 用户审核通过 → 产出 Outline JSON。

---

## 阶段五：多维功能标注

**目标**：升级 L2 输出格式，标注剧情轴/人物轴/世界轴/读者效果。

- [ ] `PlotArchitect` v2（多维标注）实现
- [ ] L2 输出格式升级：9 列表格（功能简述/剧情轴/人物轴/世界轴/读者效果/情绪/人物/说明）
- [ ] 读者效果 → L3 映射速查表

**验证**：L2 产出含多维标注的 Outline → 读者效果列可直接被 L3 消费。

---

## 阶段六：世界书管理员 Agent

**目标**：序列完成后自动更新世界书。

- [ ] 管理员 Agent 逻辑：读取产出 → 提取变化 → 更新条目 → 冲突检测 → 压缩归档
- [ ] 版本链实现：人物状态不覆写，保留 `[章节范围: 状态]` 链
- [ ] 重要性衰减：权重计算，自动归档
- [ ] 触发词自维护基础版
- [ ] 世界书管理 API
- [ ] 前端 WorldBook 页面：树形浏览器 + 条目编辑器 + 版本历史

**验证**：完成一个序列后，世界书自动更新人物状态、道具、伏笔条目。

---

## 阶段七：RAG 系统

**目标**：历史回顾检索 + 技法参考检索。

- [ ] `BaseRAGAgent` 接口
- [ ] `HybridRetriever` 实现（向量 + 关键词混合检索）
- [ ] RAG-历史回顾：序列完成后自动索引
- [ ] 双速索引：秒级关键词 + 异步向量索引
- [ ] Agent 压缩：检索结果压缩后注入上下文
- [ ] RAG-技法参考：预构建映射规则库、参考文本库
- [ ] RAG 检索 API
- [ ] L2 会议中自动插入 RAG 检索结果

**验证**：L2 会议时 RAG Agent 自动插入相关历史片段和技法参考。

---

## 阶段八：L3 叙事层

**目标**：标签化指令编排，产出细纲。

- [ ] `BaseL3Narrative` 接口
- [ ] `MappingCompilerL3` 实现：读取读者效果 → 查映射表 → 生成叙事指令
- [ ] 标签系统：标签 JSON 加载、冲突检测、协同推荐
- [ ] L3 API：`POST /scenes`、`GET /tags`
- [ ] 前端 L3Narrative 页面：指令文本区 + 标签面板（拖拽）
- [ ] 标签操作：拖入添加、点击删除、展开修改 prompt、保存组合模板

**验证**：L2 Outline 输入 → 用户拖标签编排 → 产出 ChapterPlan JSON（含视角/节奏/话语模式）。

---

## 阶段九：L4 渲染层

**目标**：细纲→正文流式生成。

- [ ] `BaseL4Renderer` 接口
- [ ] `ConstrainedRenderer` 实现：严格按 L3 约束生成
- [ ] L4 SSE API：`GET /api/projects/{id}/l4/stream`
- [ ] 前端 L4Render 页面：正文流式展示 + 用户反馈（满意/修改提示重新生成）
- [ ] 风格标签支持：L4 标签面板与 L3 相同交互模式

**验证**：L3 细纲输入 → SSE 流式生成正文 → 用户可标注不满意要求重新生成。

---

## 阶段十：前端完善与集成

**目标**：完整的前端体验。

- [ ] 项目仪表盘：项目列表、创建、进度概览
- [ ] 设置页：模块选择、LLM API 配置、提示词模板
- [ ] 章节导航：左侧树形目录、跳转
- [ ] 导入：已有小说 → 分章 → 生成世界书初稿 → 建 RAG 索引
- [ ] 导出：正文 txt/markdown、大纲、世界书
- [ ] 初始引导：创建向导（选类型→选驱动→初始化世界书）

---

## 阶段十一：打磨与测试

**目标**：跑通完整流程，修 bug。

- [ ] 端到端集成测试：L1→L4 手动跑通
- [ ] 全自动模式测试：无人值守跑通
- [ ] 错误恢复测试：SSE 断线、LLM 超时
- [ ] 边界情况处理：空世界书、无 RAG 结果、专家返回格式错误
- [ ] 性能优化：无明显卡顿
- [ ] 文档完善
