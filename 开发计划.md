# 开发计划

> 原型开发阶段，每阶段产出可运行的 MVP。后续迭代中重构。

## 阶段一：核心基础设施

**目标**：项目骨架能跑，配置能加载，LLM 能调用。

**主要文件**：`start.sh`、`requirements.txt`、`backend/main.py`、`backend/core/registry.py`、`backend/core/protocols/llm.py`、`backend/modules/llm/openai_compat.py`、`data/user/config.yaml`

- [ ] 项目初始化：`start.sh`/`start.bat`、`requirements.txt`、虚拟环境
- [ ] FastAPI 应用骨架：`backend/main.py`、CORS、基础路由
- [ ] 配置系统：加载 `data/user/config.yaml`，运行时读取
- [ ] `BaseLLMProvider` 接口 + OpenAI 兼容实现
- [ ] 模块注册表：扫描 `backend/modules/` 和 `meeting_protocols/`，发现 ABC 子类
- [ ] 基础测试框架搭建

**验证**：`./start.sh` 启动后访问 `http://localhost:7860` 返回 200。

---

## 阶段二：数据层

**目标**：项目创建、世界书读写、日志记录。

**主要文件**：`backend/core/protocols/worldbook.py`、`backend/modules/worldbook/st_style.py`、`backend/api/projects.py`、`data/projects/`

- [ ] 项目管理：`project.json` 读写，创建/列表/配置更新
- [ ] 世界书基础：`BaseWorldBook` 接口 + ST 风格实现（JSON + 触发词 + 读写）
- [ ] 版本控制：`commit()`/`revert()` 基础实现
- [ ] SQLite 日志：meeting.db 建表，会议记录写入/查询
- [ ] `data/` 目录结构初始化
- [ ] 项目 API：`POST /api/projects`、`GET /api/projects/{id}`、`PUT config`

**验证**：通过 API 创建项目，写入世界书条目，`commit()` 保存，`revert()` 回滚。

## 阶段三：L1 种子层

**目标**：用户输入创意，产出故事愿景。

**主要文件**：`backend/core/protocols/l1.py`、`backend/modules/l1/guided_form.py`、`backend/api/l1.py`、`frontend/src/pages/L1Seed.vue`

- [ ] `BaseSeedGenerator` 接口
- [ ] `GuidedFormGenerator` 实现：引导式表单 → 填充模板输出 VisionDocument
- [ ] L1 API：`POST /api/projects/{id}/l1/generate`
- [ ] 前端 L1Seed 页面：表单 + AI 辅助展开 + 可编辑产出

**验证**：填写核心梗/阅读契约/粗略大纲 → 产出 VisionDocument JSON。

## 阶段四：L2 专家会议（核心）

**目标**：三专家群聊式讨论，产出精修大纲。

**主要文件**：`backend/core/protocols/expert.py`、`meeting.py`、`l2.py`、`backend/modules/l2/expert_meeting.py`、`backend/modules/l2/experts/`、`backend/meeting_protocols/`、`backend/api/l2.py`、`frontend/src/pages/L2Meeting.vue`、`frontend/src/components/chat/`

- [ ] `BaseExpert` 接口
- [ ] `PlotArchitect` v1（Propp 功能）实现
- [ ] `WebEditor` 实现
- [ ] `CharacterDesigner` 实现
- [ ] `BaseMeetingProtocol` 接口
- [ ] `plot_driven` 协议实现
- [ ] `character_driven` 协议实现
- [ ] `market_driven` 协议实现
- [ ] `ExpertMeetingL2` 编排器：按协议调用专家、处理迭代
- [ ] `human_feedback` 回调机制：`waiting_user` SSE 事件 + 前端决策按钮
- [ ] L2 SSE 流式 API：`GET /api/projects/{id}/l2/stream`
- [ ] L2 操作 API：`POST /l2/message`、`/l2/decide`、`/l2/continue`
- [ ] 前端 L2Meeting 页面：群聊 UI（角色颜色+头像、发言卡片、用户输入、决策按钮、右侧大纲预览）
- [ ] 协作模式切换（semi_auto/full_auto/manual）
- [ ] 错误处理：超时重试、显示错误卡片

**验证**：L1 产出输入 L2 → 三位专家按顺序发言 → 用户审核通过 → 产出 Outline JSON。
