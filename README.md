# AI辅助网文创作系统 - 四层模型

## 概述

本系统将网文创作过程工程化，分为四层，每层有明确的输入、输出和职责。

```
L1 种子层 → L2 架构层 → L3 叙事层 → L4 渲染层
   ↓            ↓            ↓            ↓
 故事愿景    精修大纲      章纲/细纲      正文
```

## 文件结构

```
talk/
├── README.md              # 本文件（总览）
├── .claude/skills/        # 技能目录（Claude Code / OpenCode 自动识别）
│   ├── L1-seed/SKILL.md       # L1 种子层
│   ├── L2-architect/SKILL.md  # L2 剧情架构师
│   ├── L2-editor/SKILL.md     # L2 网络编辑
│   ├── L2-character/SKILL.md  # L2 人物设计师
│   ├── L3-narrative/SKILL.md  # L3 叙事层
│   └── L4-render/SKILL.md     # L4 渲染层
├── L1.md              # L1 详细文档（参考用）
├── L2_base.md         # L2 基座（参考用）
├── L2_architect.md    # L2 架构师详细（参考用）
├── L2_editor.md       # L2 编辑详细（参考用）
├── L2_character.md    # L2 人物详细（参考用）
├── L3.md              # L3 详细文档（参考用）
└── L4.md              # L4 详细文档（参考用）
```

## 各层说明

### L1 种子层

| 项目 | 说明 |
|------|------|
| 角色 | 创作顾问 |
| 输入 | 用户模糊创意 |
| 输出 | 《故事愿景文档》 |
| 文件 | L1.md |

**产出内容**：核心梗、阅读契约、粗略大纲、核心设定

### L2 架构层

| 项目 | 说明 |
|------|------|
| 角色 | 专家会议（三专家） |
| 输入 | 《故事愿景文档》 |
| 输出 | 《精修故事大纲》 |
| 文件 | L2_base.md + L2_architect.md + L2_editor.md + L2_character.md |

**三专家**：
- 剧情架构师：功能/序列拆解，逻辑把关
- 网络编辑：爽点/节奏把控，毒点规避
- 人物设计师：行动元分配，人设审核

**三种驱动模式**：
| 模式 | 适用 | 加载顺序 |
|------|------|----------|
| 人物驱动流 | 文青文、种田文 | character → architect → editor |
| 剧情驱动流 | 无限流、系统文 | architect → character → editor |
| 市场驱动流 | 跟风热点、商业定制 | editor → architect + character |

### L3 叙事层

| 项目 | 说明 |
|------|------|
| 角色 | 翻译器（映射编译） |
| 输入 | 《精修故事大纲》 |
| 输出 | 《章纲/细纲》 |
| 文件 | L3.md |

**核心映射**：
- 情绪 → 视角（压抑=反派内聚焦，爆发=外聚焦）
- 情绪 → 节奏（压抑=慢速扩述，爆发=中速等述）
- 场景 → 话语模式

### L4 渲染层

| 项目 | 说明 |
|------|------|
| 角色 | 执行者 |
| 输入 | 《章纲/细纲》 |
| 输出 | 正文文本 |
| 文件 | L4.md |

**执行约束**：视角、节奏、话语模式、字数

## 使用方法

### 技能系统（推荐）

Claude Code 和 OpenCode 支持 **Skills 系统**，技能按需加载，不会一次性占用所有 context。

**关键特性**：
- 每个技能都有 `description`，AI 看到技能列表
- 设置了 `disable-model-invocation: true`，只有用户手动调用
- 调用时才加载完整内容

### 完整工作流

```bash
# 在 talk/ 目录下启动 Claude Code 或 OpenCode
cd /home/ssjk/talk
claude  # 或 opencode

# Step 1: 种子层
/L1-seed
> 输入你的创意
> 获得《故事愿景文档》

# Step 2: 架构层（三专家可按需调用）
/L2-architect    # 剧情架构师：拆解序列
/L2-editor       # 网络编辑：评估爽点
/L2-character    # 人物设计师：审核人设
> 获得《精修故事大纲》

# Step 3: 叙事层
/L3-narrative
> 获得《章纲/细纲》

# Step 4: 渲染层
/L4-render
> 获得正文
```

### 驱动模式选择

根据故事类型，L2 专家调用顺序不同：

| 模式 | 适用 | 调用顺序 |
|------|------|----------|
| 人物驱动流 | 文青文、种田文 | L2-character → L2-architect → L2-editor |
| 剧情驱动流 | 无限流、系统文 | L2-architect → L2-character → L2-editor |
| 市场驱动流 | 跟风热点 | L2-editor → L2-architect + L2-character |

### 注意事项

- **技能不会自动加载**：需要用户用 `/技能名` 手动调用
- **可回退**：任何层级发现问题，可返回上层重新调用
- **L2 可迭代**：三个专家可以多次调用，直到方案满意

## 技能系统原理

### 为什么不会分散注意力？

```
┌─────────────────────────────────────────────────────────┐
│ AI 看到的（始终在 context）：                              │
│                                                         │
│ <available_skills>                                      │
│   <skill>                                               │
│     <name>L1-seed</name>                                │
│     <description>网文创作第一层 - 种子层...</description>│
│   </skill>                                              │
│   <skill>                                               │
│     <name>L2-architect</name>                           │
│     <description>网文创作第二层 - 剧情架构师...</description>│
│   </skill>                                              │
│   ...                                                   │
│ </available_skills>                                     │
└─────────────────────────────────────────────────────────┘
                           ↓
                  用户调用 /L1-seed
                           ↓
┌─────────────────────────────────────────────────────────┐
│ 此时才加载 L1-seed/SKILL.md 的完整内容                    │
│ （其他技能内容不加载）                                     │
└─────────────────────────────────────────────────────────┘
```

**关键设计**：
- `disable-model-invocation: true` → 只有用户手动调用才加载
- 每个技能末尾提示下一步 → 引导用户流程
