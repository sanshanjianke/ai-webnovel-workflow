## 一、创作模型

传统 AI 写作的根本问题：用户输入模糊创意，指望 AI 一步到位写出正文。这相当于跳过蓝图直接砌砖——没有图纸，AI 只能盲目堆砌。

本系统将创作解耦为多个层级，自顶向下、逐步细化。核心第一性原理是**"故事"与"叙事"分离**——L2 管"讲什么"，L3 管"怎么讲"。

```
L1   种子层      故事愿景     "我要写一个什么样的故事"
编排层           卷纲→章纲    "分成几卷 + 每章功能序列"
L3   叙事层      带标签草稿    "草稿嵌入叙事指令"
L4   渲染层      精修正文     "扩展打磨成终稿"
```

### L1 种子层

| 项目 | 说明 |
|------|------|
| 角色 | 创作顾问 |
| 输入 | 用户模糊创意 |
| 输出 | 《故事愿景文档》(VisionDocument) |
| 实现 | 引导式表单 + 创意对话 |

**核心功能**：

- **创意对话**：与 AI 对话梳理创意，支持流式输出和 Think 模式
- **表单填写**：结构化填写故事要素（核心梗、阅读契约、大纲等）
- **愿景生成**：把表单内容提交给 AI，生成完整的愿景文档（Markdown 格式）
- **Skill 功能**：可查看和自定义生成提示词
- **保存功能**：表单和愿景文档都可保存到文档库

**数据流**：
```
表单内容 → 合并成提示字符串 → AI 生成 → Markdown 愿景文档
```

### 编排层（原 L1.5 + L2 合并）

L1.5（卷纲规划）和 L2（章纲设计）已合并为统一的**编排层**。专家可在画布上自由拖拽组合，预设模板覆盖所有使用场景。

**分布式工作节点模型**：每个节点（专家/群聊）是一个独立 Worker，维护自己的任务队列，并行运行。任务对象在 DAG 中沿连线流动——节点完成处理后从自己队列删除，塞到下游队列。节点不需要全局调度，只看自己的队列。

| 专家 | 职责 | 核心概念 |
|------|------|---------|
| **资深作者** | 创作方向把控 | 热点判断、毒点预警、分卷方案 |
| **读者代表** | 体验审核 | 模拟读者情绪、检测疲劳点，只提问题不提供方案 |
| **剧情架构师** | 故事骨架 | 功能→序列→情节拆解，因果闭环 |
| **网络编辑** | 市场效果 | 黄金三章、压抑-释放、毒点规避 |
| **人物设计师** | 人设一致 | 行动元分配、扁形/圆形人物、OOC 检测 |
| **章节拆分师** | 卷→章翻译 | 将卷纲拆为具体章节号及概要 |
| **讨论总结师** | 讨论收敛 | 提炼共识、标注分歧 |

**三种驱动模式**：

| 模式 | 发言顺序 | 适用场景 |
|------|---------|---------|
| 人物驱动流 | 人物 → 剧情 → 编辑 | 文青文、种田文 |
| 剧情驱动流 | 剧情 → 人物 → 编辑 | 无限流、系统文 |
| 市场驱动流 | 编辑 → 剧情 ↔ 人物 | 跟风热点、商业定制 |

**协作模式**：semi_auto（专家发言后暂停）/ full_auto（全自动）/ manual（用户点名）

**多维功能标注**：每个叙事单位标注四轴——剧情轴（推进/转折/铺垫/阻碍/回收/承接）、人物轴（揭示特性/触发成长/确立关系等）、世界轴（展示设定/营造氛围等）、读者效果（期待感/满足感/代入感/爽感/惊喜感，标↑↓强度）。L2 标注好读者效果，L3 直接查映射表——省去 L3 最不可靠的"推断读者效果"这一步。

**管道模式 v2（对象管道）**：

- **PipelineObject**：任务对象在 DAG 节点间流动，每经过一个节点就累积该节点的产出（报告 + 聊天记录）
- **Agent 迭代**：专家不再是单次发言，而是多轮内部迭代——生成意见 → 自我审视 → 修正定稿（可配置中止条件：标签`<stop>` / 用户手动 / 楼层上限）
- **群聊**：多专家放入群聊容器，支持顺序循环或 @ 提及驱动发言，可配退出条件（共识 / 多数赞同 / 门禁专家）
- **断点恢复**：每个对象完成后自动保存 checkpoint；中断后可从中断位置继续，不完整轮次自动丢弃
- **ZIP 打包导出**：完整管道产出打包为 ZIP，可下载归档或拖入前端回放

**前端页面**：

- **编排画布**（Orchestration.vue + OrchestrationCanvas.vue）：Vue Flow 画布，拖拽专家/群聊/输入源/输出节点，连线定义 DAG，配置节点中止条件、读取范围、群聊参数
- **聊天弹窗**（ChatPopup.vue）：三栏布局——文档库(左) | Agent 迭代日志+输入框(中) | 文件队列+报告预览(右)。用户可向 Agent 发送反馈，采纳并停止
- **输出页面**（OutputPage.vue）：管道完成后双击输出节点打开，列出所有对象、专家链路、文件统计，支持 ZIP 下载和拖入 ZIP 加载
- **对象查看页**（ViewerPage.vue）：查看单个对象的完整迭代聊天记录，切换专家视角，支持单对象模式和 ZIP 模式

### L3 叙事层

| 项目 | 说明 |
|------|------|
| 角色 | 映射编译器 |
| 输入 | 精修大纲（含多维功能标注） |
| 输出 | 章纲/细纲（含视角、节奏、话语模式指令） |
| 实现 | MappingCompiler，规划升级为 TaggedDraftGenerator |

**核心创新——效果到技术的确定性映射**：

网文理论是"效果学"（欲扬先抑、装逼打脸），叙事学是"结构学"（外聚焦、慢速扩述）。L3 充当编译器，将效果指令翻译为技术指令：

| 效果目标 | 视角选择 | 节奏控制 | 话语模式 |
|---------|---------|---------|---------|
| 压抑/铺垫 | 反派内聚焦 | 慢速扩述 | 大量对话+心理描写 |
| 爆发/打脸 | 外聚焦 | 中速等述 | 短句动词密集 |
| 余韵/满足 | 自由间接引语 | 快速概述 | 震惊反应+留白 |

**5 种技能**：初稿生成 / 迭代修正 / 换方向重写 / 保守写作 / 加速模式

### L4 渲染层

| 项目 | 说明 |
|------|------|
| 角色 | 执行者 |
| 输入 | L3 章纲指令 |
| 输出 | 精修正文 |
| 实现 | ConstrainedRenderer（SSE 流式） |

L4 不是自由创作，而是**带约束的文本扩展**。执行约束：视角（只写所见不写所想）、节奏（字数控制）、话语模式（句式结构）。

**7 种技能**：初稿扩展 / 文风调校 / 删改压缩 / 描写增强 / 对话打磨 / 保守修复 / 终稿润色

### 迭代闭环

四层不是不可回退的单行道。L3 发现映射困难可打回编排层重审，编排层发现方向问题可重新编排。每一层 AI 提方案，人做决定。

## 二、数据支撑

长篇创作的记忆问题不能靠"全塞进上下文窗口"解决（成本 + 注意力衰减）。

```
世界书 (World Book)         RAG-历史回顾           RAG-技法参考
当前状态管理                过往记忆               写作素材
"主角现在是金丹期"          "老周第45章说过..."     "打脸场景怎么写"
```

三者分工：世界书保证**不犯错**，RAG-历史回顾保证**不遗忘**，RAG-技法参考保证**写得好**。

### 世界书

灵感来自 SillyTavern 的触发词上下文注入系统。

**四层结构**：

| 层 | 内容 | 特性 |
|----|------|------|
| 核心层 | 世界观规则、力量体系、阅读契约 | 永久条目 |
| 活跃层 | 人物状态、活跃伏笔、当前道具、势力状态 | 权重机制，随剧情更新 |
| 归档层 | 已完成序列摘要、已回收伏笔、退场人物 | 压缩为摘要 |
| 索引层 | 条目-章节映射、关系图、触发词索引 | 元数据 |

**关键机制**：
- **版本链非覆写**：`[1-30:筑基]→[31-80:金丹]→[81-今:元婴]`
- **重要性衰减**：引用 +1，10 章未引用 -1，归零归档
- **自动压缩**：已完成序列压缩为摘要
- **世界书管理员 Agent**：每序列完成后自动提取变化、更新条目、冲突检测

### RAG 系统

**双重职能**：历史回顾（检索已写章节，动态增长）和技法参考（映射规则+范例，静态库）。

**混合检索方案**（笔记7原型已验证，300 章 ~25 分钟）：
- **多维度语义切片**：同一文本按 plot/character/emotion/function 四种维度切分
- **文档增强**：每个序列生成 5 视角改写
- **混合得分**：`vector_score × 0.6 + keyword_score × 0.4`
- **双库分离**：向量库存摘要+标签（快速检索），MD 文本库存完整切片

## 三、技术栈

| 层 | 技术 |
|----|------|
| 后端 | Node.js 18+, TypeScript, Express.js |
| 前端 | Vue 3 (Composition API), Vue Flow, markdown-it |
| 实时通信 | SSE (Server-Sent Events), 流式输出 |
| 向量数据库 | Chroma（嵌入式） |
| 数据存储 | JSON (核心数据) + SQLite (日志) |
| LLM | OpenAI 兼容接口，支持 Think 模式（GLM-5/DeepSeek/Claude） |

### 核心特性

- **流式输出**：所有 AI 生成都支持流式显示，实时反馈
- **Think 模式**：支持 AI 思考过程展示（需模型支持）
- **分布式流水线**：每个节点独立 Worker + AsyncQueue，节点间通过任务对象传递实现并行处理
- **插件化架构**：模块通过装饰器自动注册
