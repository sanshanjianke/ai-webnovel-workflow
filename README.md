# AI辅助网文创作系统 - 四层模型

## 概述

本系统将网文创作过程工程化，分为**创作模型（四层）**和**数据构建流程（三道过滤）**两大体系。

```
【创作模型】
L1 种子层 → L2 架构层 → L3 叙事层 → L4 渲染层
    ↓            ↓            ↓            ↓
  故事愿景    精修大纲      章纲/细纲      正文

【数据构建流程】
数据海 → 第一道过滤 → 第二道过滤 → 第三道过滤 → 向量数据库
(400G)   (质量筛选)    (标注摘取)    (检验)      (L3+L4库)
```

## 一、创作模型（四层）

### L1 种子层

| 项目 | 说明 |
|------|------|
| 角色 | 创作顾问 |
| 输入 | 用户模糊创意 |
| 输出 | 《故事愿景文档》 |

**产出内容**：
- 核心梗/脑洞
- 阅读契约（目标读者、核心爽点、风格基调）
- 粗略大纲
- 核心设定（世界观、主角人设、金手指）
- 热点/潮流元素（可选）

### L2 架构层

| 项目 | 说明 |
|------|------|
| 角色 | 专家会议（三专家） |
| 输入 | 《故事愿景文档》 |
| 输出 | 《精修故事大纲》 |

**三专家**：

| 专家 | 职责 | 核心概念 |
|------|------|---------|
| 剧情架构师 | 构建故事骨架 | 功能/序列拆解，逻辑把关 |
| 网络编辑 | 把控市场效果 | 爽点/节奏把控，毒点规避 |
| 人物设计师 | 确保人物合理 | 行动元分配，人设审核 |

**三种驱动模式**：

| 模式 | 适用 | 顺序 |
|------|------|------|
| 人物驱动流 | 文青文、种田文 | 人物 → 剧情 → 编辑 |
| 剧情驱动流 | 无限流、系统文 | 剧情 → 人物 → 编辑 |
| 市场驱动流 | 跟风热点、商业定制 | 编辑 → 剧情 ↔ 人物 |

**当前困境**：L2需要的爽点公式、编辑经验等"方法论"知识，无法从网文文本中直接提取，暂时搁置。

### L3 叙事层

| 项目 | 说明 |
|------|------|
| 角色 | 翻译器（映射编译） |
| 输入 | 《精修故事大纲》 |
| 输出 | 《章纲/细纲》 |

**核心映射**：
- 情绪 → 视角（压抑=反派内聚焦，爆发=外聚焦）
- 情绪 → 节奏（压抑=慢速扩述，爆发=中速等述）
- 场景 → 话语模式

### L4 渲染层

| 项目 | 说明 |
|------|------|
| 角色 | 执行者 |
| 输入 | 《章纲/细纲》 |
| 输出 | 正文文本 |

**执行约束**：视角、节奏、话语模式、字数

## 二、数据构建流程（三道过滤）

### 输入：数据海

- 内容：原始小说库（txt格式）
- 规模：400G - 100G
- 说明：未经处理的原始文本，丢进Dify/LangChain处理

### 第一道过滤：质量筛选

- Agent：质量评估师 + 标签标注师
- 输出：精选书单 + 多维标签
- 规模：筛选至约50G-20G
- 标签类型：类型标签、套路标签、风格标签、平台标签

### 第二道过滤：标签标注 + 文本摘取

- Agent：序列识别师 + 叙事标注师 + 片段提取师
- 输出：带叙事符号的正文片段数据集
- 规模：筛选至约10G-5G
- 方案：并行方案更合理（L3和L4各自专注领域）

### 第三道过滤：检验

- Agent：一致性检验师 + 完整性检验师 + 质量检验师
- 输出：最终数据集

### 输出：向量数据库

1. **L3映射关系库**：效果标签 → 叙事指令 + 参考片段
2. **L4微观技法库**：技法标签 + 参考文本

**技术实现**：Dify / LangChain + 向量数据库（Milvus/Chroma）

## 三、RAG vs 微调

### RAG的困境

| 问题 | 说明 |
|------|------|
| 上下文挤占 | 检索片段占用context窗口 |
| 注意力分散 | 在参考文本和当前任务间切换 |
| 检索噪音 | 不相关片段干扰判断 |
| 知识外挂 | 无法内化成"直觉" |

### 微调成本估算（AutoDL价格）

| 训练类型 | 7B模型成本 | 时间 | 适用场景 |
|---------|-----------|------|---------|
| QLoRA微调 | ~100元 | ~10小时 | 个人 |
| LoRA微调 | ~500-1000元 | ~50小时 | 个人 |
| SFT监督微调 | ~1000-3000元 | ~20小时 | 小团队 |
| 增量预训练（10B Token） | ~4000-10000元 | ~100小时 | 小团队 |
| 增量预训练（100B Token） | ~4-6万元 | ~300小时 | 专业团队 |
| 完整训练（四阶段） | **~7-15万元** | **~500-1000小时** | 专业团队 |

### 混合策略

- **LoRA微调**：注入风格/语感/爽点直觉
- **RAG**：补充具体概念、范例、规则查询

## 五、使用方法

### 技能系统（推荐）

Claude Code 和 OpenCode 支持 **Skills 系统**，技能按需加载。

### 完整工作流

```bash
# Step 1: 种子层
/L1-seed
> 输入你的创意
> 获得《故事愿景文档》

# Step 2: 架构层（三专家按需调用）
/L2-architect    # 剧情架构师：拆解序列
/L2-editor       # 网络编辑：评估爽点
/L2-character    # 人物设计师：审核人设
> 获得《精修故事大纲》

# Step 3: 叙事层
/L3-narrative
> 获得《章纲/细纲》

# Step 4: 渲染层
/L4-render
> 获得正文
```
